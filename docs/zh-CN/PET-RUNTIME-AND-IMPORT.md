# 网页宠物运行设置与外观导入

## 3D 云灵加载开关

Side Panel 的“网页宠物性能与闲时动作”卡片提供“加载 3D 云灵”开关。

- 开启：网页宠物按需加载 `<yk-pet>`、TresJS 和 WebGL 渲染器。
- 关闭：当前页面立即销毁已有 3D 渲染器和 WebGL Canvas，只保留可点击的轻量云灵入口；页面审计、宠物记忆、Network Lab 和 Local Agent 不受影响。
- 再次开启：扩展重新按需创建 3D 渲染器，并恢复当前已保存的外观配方。

设置仅保存在 `chrome.storage.local`，不会上传，也不会新增扩展权限。

## 闲时动作

“自动播放闲时动作”是总开关。关闭后，自动调度的动作不会进入 3D 渲染器，但从网页宠物动作菜单手动点击的动作仍可播放。

每个可自动播放动作都可以单独勾选。推荐默认值保留轻量和生活动作，高能动作与彩蛋动作默认不自动播放，以降低复杂网页中的 CPU/GPU 峰值。用户仍可手动启用空翻、甩尾旋风、能量爆发、烟花等动作。

## Studio 外观导入

导入 Studio JSON 后，扩展会根据完整配方创建模型，不再把颜色、肚皮和标志简单叠加到默认云狐上。以下通道会参与模型替换：

- 身体形状、宽高深比例和头部比例；
- 耳朵、眼睛、鼻子、嘴巴；
- 前爪样式、四肢长度与间距；
- 触角杆、触角端点、间距和长度；
- 分段尾巴、尾根、方向和尾尖发光；
- 肚皮、能量核心、胸背标志和可选身体轨道。

默认内置配方仍使用原有完整动作模型；Studio 或导入配方使用完整配方模型，二者不会同时叠加。

## 人工验收

1. 在 Side Panel 关闭“加载 3D 云灵”，确认页面中的 Canvas/WebGL 消失，但轻量入口仍可点击并打开菜单。
2. 重新开启 3D，确认无需刷新即可恢复宠物。
3. 关闭闲时动作并保持页面静置至少 30 秒，确认宠物不自动播放动作；手动点击动作仍正常。
4. 只勾选“招手”，静置后确认不会自动播放未勾选动作。
5. 导入一份与默认模型差异明显的 Studio JSON，确认身体、头部、眼睛、耳朵、触角、四肢和尾巴整体变化，而不只是多出一个标志或肚皮。
6. 在 Chrome 任务管理器中观察关闭 3D 前后的 GPU/内存变化；此项需要真实 Chrome/WebGL 环境人工完成。
