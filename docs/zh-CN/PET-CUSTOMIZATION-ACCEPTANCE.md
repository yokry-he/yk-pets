# 宠物可配置系统人工验收

自动 CI 不能替代真实 WebGL、颜色辨识和 Chrome 扩展同步检查。

## 鼻子和嘴巴

- 依次选择五种鼻子，确认正面轮廓、厚度或材质有明显区别；
- 依次选择五种嘴巴，确认微笑、猫系、线条、张嘴和嘟嘴可直接辨认；
- 播放说话、吃饭、开心和喷嚏动作，确认嘴部仍跟随动作；
- 分别改变口鼻部、鼻子、嘴巴、舌头和脸颊颜色。

## 肚皮

- 恢复默认后确认肚皮为椭圆形；
- 逐一检查十种形状，并确认名称与轮廓一致；
- 调整颜色、宽高、左右位置、上下位置、旋转和柔和度；
- 在椭圆体、梨形、豆形和圆角方糖身体上检查贴合；
- 导入旧 `model-default` 配方，确认迁移为椭圆形；
- 导入旧自定义盾牌、豆形、水滴和爱心配方，确认映射正确。

## 颜色和范围

- 修改每个颜色通道并确认预览立即更新；
- 确认 UI 明确标注身体/尾根、四肢/尾中段和爪部/尾尖暖色的兼容关联；
- 将身体、头部、四肢、耳眼、尾巴、触角和肚皮调整到新边界；
- 保存、刷新 Studio，再确认所有值保留；
- 导出 JSON 后重新导入，确认颜色和扩展参数不丢失；
- 超出旧推荐范围后检查页应提示风险，而不是静默恢复旧值。

## 交互与扩展

- 连续拖动一个滑杆后只需撤销一次即可恢复拖动前值；
- 直接输入数值、单项重置、撤销和重做都应工作；
- 刷新未正式保存的 Studio，确认本地草稿恢复；
- 点击“保存并用于同步”，再在 Chrome 扩展中导入或同步外观；
- 确认扩展和 Studio 的鼻嘴、肚皮及颜色一致；
- 使用浅色、深色和网页背景以及四个视角检查可读性；
- 不得观察到新增权限、数据上传或后台轮询。
