# 宠物记忆

宠物记忆是 YK-PETS 的本地优先网页上下文收件箱。它不是独立便签，而是把网页想法、选中文字、页面审计问题和后续任务连接到云灵与 Local Agent 工作流。

## 功能入口

可以通过以下方式进入：

- Side Panel 顶部的“宠物记忆”工作区；
- 网页宠物的“功能 → 宠物记忆”；
- 当前页面存在记忆时，点击网页宠物左上方的记忆数量徽章；
- Chrome 快捷键：Windows/Linux 为 `Ctrl+Shift+.`, macOS 为 `Command+Shift+.`；
- 选中文字后右键“交给云灵记住”；
- 页面空白处右键“在宠物记忆中记录此页”；
- 页面审计问题卡上的“记住”。

快捷键可以在 `chrome://extensions/shortcuts` 中重新配置。

## 快速记录

编辑器会自动带入当前页面标题和地址。选中文字后打开编辑器时，还会显示原文预览。

一张记忆卡可以包含：

- 标题与正文；
- 页面标题与 URL；
- 选中文字；
- 标签；
- 低、中、高优先级；
- 收件箱、待办、进行中、已完成或已归档状态；
- 关联的审计问题、网络请求或补丁标识。

在编辑器中按 `Command/Ctrl + Enter` 可以立即保存。

## 工作流

默认流程为：

```text
收件箱 → 待办 → 已完成
```

- 新记录默认进入收件箱；
- “加入待办”把内容转为任务；
- “标记完成”触发云灵的完成反馈；
- 已完成任务可以恢复为待办；
- 归档后 6 秒内可以撤销。

顶部筛选支持收件箱、待办、已完成和当前页面。还可以按标题、正文、来源页面和标签搜索。

## 与 3D 宠物的互动

- 保存记忆后，云灵执行招呼动作并确认已经记住；
- 完成任务后，云灵进入庆祝反馈；
- 当前页面存在未归档记忆时，网页宠物左上方显示可点击的记忆数量徽章；
- 从宠物菜单打开记忆时，Side Panel 会直接切换到对应工作区；
- 选中文字右键保存后，网页宠物会即时确认；
- JSON 导入完成后，云灵只给出一次短促结果反馈。

这些反馈不会自动打开大型遮挡层，也不会改变网页布局。

## 当前页面记忆徽章

记忆数量徽章是独立按钮，不嵌套在宠物拖动按钮中，可以使用鼠标、`Tab`、`Enter` 或空格操作。点击后会：

- 打开或切换到 Side Panel 的“宠物记忆”工作区；
- 清除上次遗留的搜索词和标签筛选；
- 直接激活“当前页面”视图，并把键盘焦点放到该筛选按钮；
- 在 Side Panel 中显示加载、成功或失败状态；
- 不同时打开宠物环形菜单，也不影响宠物拖动。

网页与 Side Panel 之间只写入一条最多保留 30 秒的一次性本地请求。请求只包含规范化后的页面 URL、标题和创建时间，消费后立即删除，不轮询、不上传，也不新增权限。如果用户在打开过程中切换了活动标签页，Side Panel 会停止应用请求并提示重试，避免把筛选错误地应用到其他页面。

## 页面摘录重新定位

包含选中文字的记忆卡会显示“定位摘录”。点击后：

- 如果当前标签页就是来源页面，直接在当前页面查找；
- 如果当前页面不同，会由用户动作打开来源页面后再定位；
- 如果卡片已有元素 selector，会先缩小范围；否则直接进行页面全文匹配；
- 使用精确文字选区和短时覆盖层高亮，页面有多处相同文字时会报告匹配数量；
- 页面内容变化、来源地址无效或找不到可见匹配时会给出明确失败状态；
- DOM 扫描只在点击“定位摘录”后执行，并限制最多 20,000 个文本节点和约 2 MB 文本；
- 高亮 12 秒后自动收起，不注入永久标记，不轮询页面，不上传摘录。

没有 selector 的旧记忆卡会直接使用保存的原文重新查找；导入或其他功能生成的卡片如果带有 selector，也会自动利用。

## 审计问题转记忆

页面审计的问题卡提供“记住”操作。生成的记忆卡会：

- 使用问题标题和说明；
- 保存页面地址和元素 selector；
- 根据问题严重程度设置优先级；
- 默认进入待办；
- 自动添加“页面审计”和问题类别标签；
- 保存稳定的 `relatedAuditIssueId`，用于之后恢复同一个审计问题。

### 从审计记忆继续生成源码补丁

Audit 类型记忆卡会显示“生成补丁”。点击后，YK-PETS 会：

1. 从本地记忆存储重新读取该卡片，并使用 `relatedAuditIssueId`；
2. 确认当前活动标签页仍是记忆卡的来源页面；
3. 读取当前标签页对应的本地审计报告 `nova:report:<tabId>`；
4. 切换到“页面审计”，清除严重程度和问题类别筛选；
5. 使用 IssueCard 的稳定 `data-issue-id` 精确定位、滚动并聚焦对应问题；
6. 点击现有“源码补丁”按钮，继续进入原有 Local Agent 流程。

这条入口不会自行创建 WebSocket，也不会自行发送 `patch.generate` 或 `patch.apply`。未连接时仍由现有 `ensureAgentConnected()` 打开连接设置并验证地址、口令与项目；连接中或已有补丁操作进行时会阻止重复触发。后续的 Diff、Apply、Checks 和 Rollback 完全沿用原有安全控制。

如果当前标签页已经变化、审计报告不属于当前页面、关联问题已从报告中消失，或记忆卡缺少关联标识，Side Panel 会显示明确错误，不会猜测标题或模糊匹配其他问题。

## 导入与导出

宠物记忆工具菜单支持：

- 导入 JSON：恢复或迁移由 YK-PETS 导出的 v1 记忆文件；
- 导出 Markdown：适合复制到文档、Issue 或任务管理工具；
- 导出 JSON：适合本地备份和迁移。

JSON 导入采用安全合并：

- 不清空现有记忆，也不覆盖同 ID 卡片；
- 完全重复的卡片会跳过；
- 同 ID 但内容不同的卡片会作为冲突跳过；
- 无效卡片、重复卡片、冲突和容量截断会分别显示数量；
- 支持标准 `{ version: 1, cards: [...] }` 文件，也支持直接导入卡片数组；
- 单个导入文件最大 8 MB；
- 导入仍受最多 500 张记忆卡的本地容量边界约束，超出的导入卡片不会写入。

默认导出仅包含当前未归档记忆。导入文件只在用户主动选择后读取，不会上传。

## 数据与隐私

- 记忆卡存储在扩展的 `chrome.storage.local`；
- 当前版本默认不登录、不上传、不进行云同步；
- 选中文字只在用户主动使用右键菜单或快捷记录时读取；
- 页面上下文仅保存标题、HTTP/HTTPS URL 和用户主动选择的文本；
- 当前页面徽章请求为短期本地数据，消费或过期后删除；
- 审计补丁续接只读取本地记忆卡、当前标签页和本地审计报告，不上传页面内容；
- 当前实现没有申请 `unlimitedStorage`，审计补丁续接也没有新增任何浏览器权限；
- 最多保留 500 张记忆卡，超过上限时优先保留未归档内容；
- 删除扩展会删除浏览器中该扩展的本地数据，重要内容应先导出。

## 当前限制

- 当前版本不保存网页截图；
- 页面摘录可以按需重新定位，但页面文字大幅修改后可能无法找到；
- 不提供云同步和跨设备合并；
- 只有带有效 `relatedAuditIssueId`、来源页面和当前本地审计报告的 Audit 记忆卡可以继续进入 Local Agent；普通记忆卡不会修改源码；
- JSON 导入不会覆盖冲突卡片，也不提供交互式逐条冲突合并。
