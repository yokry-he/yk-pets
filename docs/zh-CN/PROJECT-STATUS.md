# YK-PETS 当前项目状态

## 当前基线

- 稳定参照：`v0.6.10`；
- 品牌迁移分支：`agent/yk-pets-rebrand-v0610`；
- 云狐工坊分支：`agent/cloud-fox-studio-v0610`；
- 产品品牌：`YK-PETS`；

- 当前阶段：云狐外观配方、受限部件编辑和实时动作验证 MVP。

## v0.6.10 兼容策略

为避免升级破坏，下列旧技术标识暂时保留：

- `Nova*` 类型作为弃用别名；
- `NOVA_*` Wire Message 作为现有扩展通信值；
- `@nova/*` 私有 Workspace scope；
- `nova:*` Storage Key，与 `yk-pets:*` 双向镜像；
- `.nova/agent.json`，会自动迁移到 `.yk-pets/agent.json`；
- `nova-agent` 命令别名。

这些标识只承担兼容职责，用户可见界面和新功能不得继续使用 NOVA 作为品牌或宠物名字。

## 当前工作原则

- 产品品牌、宠物物种、宠物名字和外观配方必须独立建模；
- 外观配方不得复制或覆盖亲密度、记忆等个体状态；
- 四肢和骨架只允许物种声明范围内的受限调整；
- Domain 层不依赖 Vue、Chrome API、DOM 或具体 Agent Provider；
- 修复必须覆盖真实运行链路，不能只调整界面表现；
- 保持现有规则、消息和本地数据向后兼容；
- 新宠物必须通过注册定义接入，不能把物种专属动作继续写进通用核心。

## 现有稳定能力

- 3D 云狐云灵的动作、拖拽、菜单和可切换音色；
- 云狐工坊的外观配方编辑、实时 3D 预览和动作验证；
- 页面审计、规则独立选择、问题定位和可撤销预览；
- Fetch/XHR 采集、Mock、延迟和完整 JSON 响应修改；
- Token 认证的本地 WebSocket Agent；
- SHA-256 并发修改保护、显式确认、验证和回滚；
- 类型检查、专项回归脚本和生产构建门禁。

## 下一阶段方向

1. 将云狐工坊保存的外观配方接入扩展中正式运行的云灵；
2. 把物种定义、外观配方验证和渲染器接口提取为框架无关包；
3. 将宠物状态机、动作调度和闲时策略提取为 `pet-core`；
4. 增加第二个真实物种，验证部件槽位和能力降级；
5. 提供普通 JavaScript API 与 `<yk-pet>` Web Component；
6. 核心 API 稳定后再增加 React、Vue、Svelte 等薄适配层。
