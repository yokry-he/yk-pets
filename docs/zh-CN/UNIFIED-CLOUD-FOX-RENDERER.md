# 统一云狐渲染器与无损优化

## 唯一模型和动作实现

默认云灵、Studio 当前配方和导入 JSON 不再选择不同模型。三者只提供不同的配方数据，并共同使用 `apps/playground/app/components/studio/ExtensionAlignedCloudFox.vue`：

```text
默认云灵配方 ─┐
Studio 配方 ──┼─> 统一云狐模型 + 统一动作帧 + 统一特效
导入配方 ────┘
```

默认宠物因此不再拥有特殊硬编码拓扑。它作为普通的默认配方进入同一个组件，与其他配方经过相同归一化、部件装配和动作路径。

## 高空烟花秀

高空烟花秀以此前 Chrome 扩展正式实现为唯一标准，并固化到统一云狐组件的唯一动作运行时：

- 12 秒完整时长；
- 三次连续高空发射；
- 每次爆炸 48 个粒子；
- 保留原有四组精选配色和四种爆炸花型；
- 保留火箭、拖尾、错峰粒子、重力下坠和衰减曲线；
- 宠物抬头追踪爆炸位置；
- 双眼、触角和左右前爪随当前发射轮次联动。

Studio 和扩展现在调用同一个 Vue/TresJS 组合组件；烟花随机种子、发射原点、花型和配色来自同一共享算法，头部、双眼和粒子组件消费同一轮次计划。旧 Studio 烟花对象池和算法已经从通用特效层删除，只有这一个正式烟花组件会被创建和更新。

## 不降低视觉质量的优化

本阶段不修改以下视觉参数：

- compact DPR `[0.75, 1]`，普通 DPR `[0.9, 1.25]`；
- compact 30 FPS，普通 40 FPS；
- 双点光源、相机、FOV、抗锯齿策略；
- 星云光晕、材质、模型细分和烟花粒子数量。

优化只减少用户看不到时的无效工作：

1. 标签页隐藏时停止 TresJS 渲染循环；
2. 宠物离开视口时由 `IntersectionObserver` 暂停动作更新；
3. 恢复后通过统一动作令牌干净地重新开始当前动作，避免把后台停留时间算入进度后直接跳到结尾；
4. 烟花方向、轮次计划与固定 48 粒子对象复用，避免每次发射重新创建粒子或在头部、眼睛和特效中复制算法；
5. 用户关闭 3D 时仍彻底销毁 `<yk-pet>` 和 WebGL 场景。

## 人工验收

自动 CI 不能替代真实 Chrome/WebGL 视觉验收。至少检查：

1. 默认云灵与导入经典云灵配方在四视角下结构一致；
2. Studio 同一配方与扩展的身体、头部、耳朵、眼睛、触角、四肢和尾巴一致；
3. Studio 与扩展的高空烟花秀在相同动作阶段具有相同发射、爆炸、抬头、视线和举爪逻辑；
4. 切换后台再返回时动作不会突然跳到结尾，并从当前动作的稳定起点重新播放；
5. 开启 3D 时画质、灯光、光晕和粒子密度没有下降。
