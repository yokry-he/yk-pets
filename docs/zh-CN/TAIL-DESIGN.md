# NOVA 云狐尾巴设计方案

## 1. 当前问题

旧版尾巴使用单个 Torus 圆环几何体，虽然实现简单，但存在以下问题：

- 轮廓接近塑料圆管，不像狐狸尾巴。
- 根部、腰部和尾尖粗细基本一致，缺少自然渐缩。
- 整条尾巴只能整体旋转，动作像刚性零件。
- 发光尾尖与主体之间缺少材质过渡。
- 从正面观察时，圆环容易与身体轮廓竞争，降低角色识别度。

## 2. 推荐方案：分段柔性云尾

v0.2.4 采用三段式柔性云尾：

1. **根部段**：较粗、低饱和，承担与身体的连接。
2. **中间段**：逐渐变细并向上弯曲，形成狐狸尾巴的蓬松 S 形轮廓。
3. **能量尾尖**：使用青紫色发光材质，末端增加柔和能量球与低透明外辉光。

每一段使用独立的 Catmull-Rom 曲线和 TubeGeometry，而不是一个完整圆环。中段和尾尖采用嵌套关节，因此可以产生延迟摆动。

### 动作规则

- 根部先移动，中段延迟约一个相位，尾尖再次延迟。
- 开心、玩耍和跳跃状态提高摆动速度与幅度。
- 思考状态降低根部幅度，但增强尾尖能量脉冲。
- 休息和睡眠状态让尾巴向身体方向轻微收拢。
- 尾尖的 Y 轴轻微摆动，使动作不局限在一个二维平面。

### 材质规则

- 根部：身体阴影色，粗糙度较高。
- 中段：主体毛色，形成自然亮度过渡。
- 尾尖：青色能量材质，带 Emissive。
- 外辉光：低透明紫色球体，不使用图片纹理。

## 3. 备选方案 A：液态光带尾

将尾巴设计为半透明 Ribbon 或 Shader 光带，适合强化“数字生命”定位。

优点：

- 视觉科技感最强。
- 可通过 Shader 做流动、噪声和数据脉冲。
- 与星云背景容易形成统一视觉语言。

缺点：

- 狐狸特征会减弱。
- Shader 和透明排序增加移动端成本。
- 与浅色网页叠加时需要额外处理对比度。

适合作为隐藏模式或高能状态，而不是默认尾巴。

## 4. 备选方案 B：双层毛绒大尾

使用一条实体蓬松尾巴，加一层较小的半透明能量内核。

优点：

- 狐狸识别度最高。
- 可爱感和角色亲和力更强。
- 适合后续替换为带骨骼的 GLB 模型。

缺点：

- 程序化几何体较难表现真正毛绒感。
- 尾巴体积较大，容易遮挡身体和功能按钮。
- 对取景、阴影和碰撞范围要求更高。

## 5. 备选方案 C：多尾形态

正常状态保留单尾，隐藏模式展开三条或五条能量尾。

优点：

- 高能状态的记忆点非常强。
- 可以把不同尾巴映射到性能、无障碍、错误、网络等能力。
- 适合品牌宣传和演示场景。

缺点：

- 默认常驻会造成视觉拥挤。
- 动作与性能成本明显上升。

建议仅在 `secretMode` 或完整审计成功时短暂触发。

## 6. 推荐演进路线

- v0.2.4：默认使用分段柔性云尾。
- v0.3.x：隐藏模式增加液态光带或三尾展开。
- v1.0：切换到骨骼 GLB 后，保留相同的根部、中段、尾尖动画语义。

这一方案在狐狸识别度、动态表现、性能和当前程序化模型的实现成本之间最均衡。
