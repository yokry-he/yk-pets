# v0.7.3 Agent 工具能力与权限治理

每个工具必须声明：ID、版本、说明、Scope、风险等级、副作用、执行位置、确认方式、输入/输出 Schema 和超时。

策略顺序：

1. 工具必须已注册；
2. 请求 Scope 必须是工具声明的子集；
3. Origin 和项目根目录必须符合工具自身边界；
4. 显式拒绝规则优先于授权；
5. Grant 必须覆盖 Subject、Tool、Scope、Origin、Project、期限和次数；
6. 高风险写入、命令执行和回滚必须使用一次性确认令牌；
7. 决策和执行结果进入审计日志。

v0.7.2 的五个公共 Local Agent 工具已经提供标准声明：连接、生成补丁、写入补丁、运行检查、回滚补丁。
