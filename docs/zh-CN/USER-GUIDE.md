## 1. YK-PETS 是什么

YK-PETS 是一个住在网页中的 3D AI 前端工程伙伴平台。默认宠物云灵会显示在普通网页右下角，并提供：

- 3D 宠物互动、动作、拖拽和语音反馈；
- 页面性能、无障碍、SEO、最佳实践和 DOM 结构审计；
- 问题定位以及可撤销的页面临时修复预览；
- Fetch/XHR 请求采集、延迟、Mock 和完整 JSON 响应替换；
- 连接本地项目 Agent，生成、确认写入、验证和回滚源码补丁；
- 宠物工坊外观编辑，以及将保存的外观同步到浏览器扩展。

请区分以下三个概念：

```text
产品品牌：YK-PETS
宠物名字：云灵 / Zeph
宠物物种：云狐 / Cloud Fox
```

更换宠物名字或增加新物种，不应改变 YK-PETS 产品品牌。

## 2. 使用模式与组成部分

YK-PETS 包含三个主要部分。

| 部分 | 主要用途 | 是否必须 |
|---|---|---|
| Chrome/Edge 扩展 | 网页内云灵、页面审计、Side Panel、Network Lab、动作语音 | 必须 |
| YK-PETS Local Agent | 读取本地项目、生成和应用源码补丁、运行检查和回滚 | 只在修改源码时需要 |
| Playground / 宠物工坊 | 编辑云狐外观、动作预览、导入导出和同步扩展 | 创建或测试宠物外观时需要 |

### 2.1 只使用页面审计

只需要安装扩展，不需要运行 Local Agent。可以完成：

- 审计页面；
- 定位问题；
- 临时预览修复；
- 使用 Network Lab；
- 播放宠物动作和语音。

### 2.2 修改本地项目源码

除扩展外，还要启动 Local Agent，并在 Side Panel 中填写连接地址和口令。

### 2.3 创建自定义宠物外观

启动 Playground，进入 `/studio` 宠物工坊。保存后，已安装的当前开发版扩展可以接收并应用配方。

## 3. 运行要求和已知限制

### 3.1 开发环境

- Node.js 22 或更高版本；
- Corepack；
- pnpm 11.13.1；
- Chrome 或 Edge，且支持 Manifest V3 Side Panel；
- 一个普通的 `http://` 或 `https://` 页面用于扩展测试。

推荐先验证：

```bash
node --version
corepack --version
```

### 3.2 浏览器限制

扩展无法注入以下页面：

- `chrome://`、`edge://` 等浏览器内部页面；
- Chrome Web Store 或 Edge Add-ons 商店页面；
- 浏览器安全策略禁止扩展脚本运行的页面；
- 某些内置 PDF、文件下载页或特殊沙箱页面。

### 3.3 音频限制

浏览器可能要求用户先点击页面或宠物，才允许播放动作语音。系统语音的最终音色还取决于操作系统已安装的中文 TTS 声线。

### 3.4 当前开发版本边界

Phase 14 已实现 `pet-core`、`<yk-pet>`、Vue 适配器和 Studio 配方同步，但仍建议在每次扩展重新构建后执行一次人工端到端检查。

## 4. 安装和加载浏览器扩展

### 4.1 安装依赖

在仓库根目录运行：

```bash
corepack enable
corepack prepare pnpm@11.13.1 --activate
pnpm install --frozen-lockfile
```

`--frozen-lockfile` 会确保安装结果与仓库中的锁文件一致。

### 4.2 构建扩展

```bash
pnpm build:extension
```

构建目录为：

```text
apps/extension/.output/chrome-mv3
```

### 4.3 在 Chrome 中加载

1. 打开 `chrome://extensions`。
2. 打开右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择 `apps/extension/.output/chrome-mv3`。
5. 将 YK-PETS 固定到浏览器工具栏，便于打开 Side Panel。
6. 刷新已经打开的测试网页。

### 4.4 在 Edge 中加载

1. 打开 `edge://extensions`。
2. 开启“开发人员模式”。
3. 点击“加载解压缩的扩展”。
4. 选择同一个 `apps/extension/.output/chrome-mv3` 目录。

### 4.5 开发监听模式

需要持续修改扩展时运行：

```bash
pnpm dev:extension
```

代码更新后仍建议：

1. 在扩展管理页点击“重新加载”；
2. 刷新目标网页；
3. 刷新已经打开的宠物工坊页面。

旧标签页不会自动获得新的 Content Script。

### 4.6 创建 ZIP

```bash
pnpm zip:extension
```

ZIP 位于 WXT 的 `.output` 目录，实际文件名和版本以 Manifest 与构建输出为准。

## 5. 第一次使用

完成扩展加载后：

1. 打开一个普通网页，例如项目的 `/audit-lab` 测试页。
2. 刷新网页。
3. 等待右下角出现云灵。
4. 单击云灵，确认环形菜单展开。
5. 双击云灵，确认开始页面审计。
6. 点击工具栏中的 YK-PETS 图标，确认 Side Panel 打开。
7. 在 Side Panel 中选择“动作语音音色”并试听。
8. 暂不需要源码修改时，可以不启动 Local Agent。

如果没有显示云灵，请先查看[常见故障排查](#19-常见故障排查)。

## 6. 网页内云灵交互

### 6.1 基础交互

| 操作 | 结果 |
|---|---|
| 单击云灵 | 展开或收起当前菜单 |
| 双击云灵 | 快速执行页面审计 |
| 右键云灵 | 打开工程工具模式 |
| 悬停 | 云灵注视指针；不会打断正在播放的主要动作 |
| 拖拽 | 调整宠物在当前网页中的位置 |
| 鼠标滚轮 | 菜单有多页时切换菜单页 |
| Esc | 关闭已经打开的菜单 |
| Tab / Enter | 键盘选择和执行菜单按钮 |

### 6.2 菜单模式

菜单分为三种模式。

#### 功能

- 页面审计；
- 打开完整报告。

#### 动作

当前网页扩展菜单包含 18 个可直接选择的动作：

- 招手；
- 跳跃；
- 扑腾；
- 休息；
- 伸懒腰；
- 吃饭；
- 空翻落地；
- 甩尾旋风 / 尾巴风车；
- 飞扑接球；
- 能量爆发；
- 害羞偷看；
- 星星杂耍；
- 云朵午睡；
- 星光喷嚏；
- 高空烟花秀；
- 好奇扫描；
- 触角聚能；
- 尾光流动。

高能动作持续时间较长。手动选择另一个动作时，当前动作会先经过短暂中性姿态再切换。

#### 工程

- 网络实验室；
- 连接 Agent；
- 生成补丁；
- 应用补丁；
- 项目验证；
- 回滚补丁。

复杂工程操作会打开 Side Panel，网页内宠物不会直接访问本地文件系统。

### 6.3 闲时动作

云灵在页面可见、菜单关闭、没有忙碌任务且系统没有要求减少动态效果时，会按不同冷却时间播放普通、生活、高能和稀有彩蛋动作。

页面切到后台时，闲时动作调度会暂停。

### 6.4 审计结果快捷控制

审计完成后，提示卡可能显示：

- 上一个问题；
- 下一个问题；
- 预览修复 / 撤销预览；
- 完整报告。

提示卡可以单独关闭，不影响宠物菜单。

## 7. Side Panel 使用说明

点击浏览器工具栏中的 YK-PETS 图标，或从云灵菜单执行需要详情的操作，即可打开 Side Panel。

### 7.1 顶部区域

顶部显示：

- 当前扩展版本；
- Agent 状态；
- 当前功能工作区。

### 7.2 宠物入口提示

入口卡说明网页内云灵的单击、双击、右键和拖拽行为。点击“呼叫”可以让云灵显示反馈。

### 7.3 动作语音音色

可以选择并试听：

- 星云外星人；
- 萌系少女；
- 萌宠伙伴；
- 静音。

### 7.4 工作区标签

当前主要工作区为：

- 页面审计；
- 网络实验室。

### 7.5 Agent 面板

打开 Agent 面板后，可以填写：

- WebSocket 地址；
- 连接口令。

连接成功后会显示项目和连接状态。

### 7.6 补丁面板

生成补丁后会显示：

- 修改原因；
- 置信度；
- 目标文件；
- Diff 或候选源码文件；
- 确认写入、运行检查和回滚按钮；
- 每个检查命令的 PASS / FAIL 与输出摘要。

## 8. 页面审计完整流程

### 8.1 选择审计范围

Side Panel 按类别展示 16 条子规则。父分类可以全选或取消分类下所有规则，也可以逐条勾选。

#### 性能

- 图片没有声明固有尺寸；
- 首屏以下图片未延迟加载；
- 页面完整加载时间偏长；
- 主线程存在长任务；
- 大型网络资源。

#### 无障碍

- 图片缺少替代文本；
- 表单控件缺少可访问名称；
- 按钮没有可访问名称；
- 链接没有可访问名称；
- 标题层级发生跳跃。

#### SEO

- 页面缺少标题；
- 页面缺少 Meta Description。

#### 最佳实践

- 页面缺少 Viewport 配置；
- HTTPS 页面引用不安全资源。

#### DOM / 结构

- 页面存在重复 ID；
- DOM 规模偏大。

未勾选的规则不会执行。如果清空了全部规则，审计按钮会禁用。

### 8.2 开始审计

可通过以下任一方式开始：

- 双击网页内云灵；
- 从云灵“功能”菜单选择页面审计；
- 在 Side Panel 点击“开始页面审计”。

从网页宠物快速审计时，会使用 Side Panel 最近保存的规则范围。

### 8.3 查看结果

审计结果包括：

- 页面健康度分数；
- DOM 节点数；
- 资源数量和传输体积；
- LCP；
- CLS；
- 高、中、低严重度问题数量；
- 本次实际执行的类别和子规则。

可以按严重度或类别筛选问题。

### 8.4 定位问题

点击问题卡或使用宠物提示卡中的上一个 / 下一个按钮。扩展会：

1. 滚动到目标元素；
2. 在独立高亮层绘制边框；
3. 显示问题名称；
4. 不改变页面布局，也不拦截目标元素点击。

### 8.5 临时预览修复

部分规则提供可逆预览：

1. 点击“预览修复”；
2. 扩展保存目标元素原有属性和样式；
3. 在当前 DOM 上应用临时变化；
4. 点击“撤销预览”恢复。

预览不会修改源码，也不会改变服务器数据。刷新页面也会清除临时预览。

### 8.6 审计结果的边界

页面健康度用于排序和解释，不是 Lighthouse、真实用户监控或完整安全扫描的替代品。可访问名称和 alt 文本建议仍需人工确认语义。

## 9. Network Lab 与 Mock

### 9.1 打开 Network Lab

可以从：

- Side Panel 的“网络实验室”标签；
- 云灵的“工程”菜单；
- 审计或工程流程中的导航入口。

### 9.2 请求采集

Network Lab 可以观察当前页面的 Fetch 和 XHR 请求，包括：

- URL 和方法；
- 状态码；
- 请求耗时；
- 资源类型；
- Query 参数；
- 请求头；
- 响应头；
- 请求体预览；
- 响应预览。

扩展重载后必须刷新目标网页，才能获得最新网络采集脚本。

### 9.3 筛选和分析

可以按以下维度筛选：

- 请求类型；
- 状态；
- 搜索词；
- 慢请求或异常请求。

展开单条请求可以查看详细字段。

### 9.4 从请求创建规则

典型流程：

1. 选择一条请求；
2. 点击从请求创建规则；
3. 设置 URL / 方法匹配条件；
4. 选择延迟、Mock 状态或响应修改；
5. 保存规则；
6. 重新触发页面请求；
7. 检查命中状态和实际响应。

### 9.5 延迟

可以为命中的 Fetch/XHR 请求增加延迟，用于模拟慢网络和观察加载状态。

### 9.6 Mock 状态、响应头和响应体

规则可以修改：

- HTTP 状态码；
- 响应头；
- 文本或 JSON 响应体。

完整 JSON 替换适合测试成功、空数据、错误和边界结构。不要在包含敏感真实数据的生产账号页面上随意启用规则。

### 9.7 站点控制

网络规则按站点启用。调试结束后应：

- 关闭站点网络能力；或
- 停用 / 删除测试规则；
- 刷新页面确认真实请求恢复。

### 9.8 影响范围

Mock 只影响当前浏览器页面中的请求处理，不会修改服务器数据库。但是页面可能根据伪造响应执行后续操作，因此测试重要业务页面时仍需谨慎。

## 10. 连接 YK-PETS Local Agent

### 10.1 何时需要 Agent

以下操作需要 Agent：

- 定位本地源码候选；
- 生成源码 Diff；
- 写入文件；
- 运行项目检查；
- 回滚最近一次由 YK-PETS 写入的补丁。

页面审计、DOM 预览和 Network Lab 不需要 Agent。

### 10.2 从仓库运行

在 YK-PETS 仓库中开发时：

```bash
pnpm dev:agent
```

### 10.3 使用构建后的 CLI

```bash
yk-pets-agent dev --root /path/to/project
```

兼容期内 `nova-agent` 仍可能作为旧命令别名存在。

### 10.4 启动信息

Agent 会显示：

- WebSocket 地址，通常为 `ws://127.0.0.1:4736`；
- 随机连接口令；
- 项目根目录；
- 项目名称；
- 框架；
- 包管理器；
- 允许执行的脚本。

### 10.5 在 Side Panel 连接

1. 打开 Agent 面板。
2. 输入 WebSocket 地址。
3. 输入连接口令。
4. 点击连接。
5. 确认状态变为已连接。

Agent 默认只监听本机回环地址，不接受局域网或互联网连接。

### 10.6 Agent 配置

配置文件为：

```text
.yk-pets/agent.json
```

它包含本地连接信息，应加入 `.gitignore`。如果只存在旧的 `.nova/agent.json`，Agent 会尝试迁移原 Token 和端口。

## 11. 生成、写入、验证和回滚源码补丁

### 11.1 前置条件

- 已完成一次页面审计；
- 当前选择了一个问题；
- Local Agent 已连接；
- Agent 指向正确的项目根目录。

### 11.2 生成补丁

1. 在问题卡点击生成源码补丁，或从云灵工程菜单选择“生成补丁”。
2. Agent 根据问题证据和源码提示搜索候选文件。
3. Side Panel 显示目标文件、原因、置信度和 Diff。
4. 置信度不足或映射不可靠时，只显示候选文件，不提供确认写入。

### 11.3 检查 Diff

确认：

- 文件路径属于当前项目；
- 修改只覆盖当前问题；
- 没有误删无关代码；
- 文本、选择器和属性语义正确；
- 没有引入远程脚本、任意命令或敏感数据。

### 11.4 确认写入

只有点击“确认写入”后才会修改源码。写入前 Agent 会：

1. 检查文件路径不能逃逸项目根目录；
2. 比对生成补丁时保存的 SHA-256；
3. 拒绝覆盖已经被其他程序修改的文件；
4. 创建可回滚备份；
5. 写入补丁。

### 11.5 运行检查

点击“运行检查”后，只允许执行项目预定义的：

```text
typecheck
test
build
```

命令使用无 Shell 拼接方式运行，不支持网页或模型传入任意命令参数。

### 11.6 回滚

### 11.7 推荐工作方式

即使全部检查通过，也建议：

- 查看 Git Diff；
- 运行项目自己的浏览器回归；
- 再决定是否提交到 Git。

## 14. 将工坊外观同步到浏览器扩展

### 14.1 前置条件

- 使用包含 Phase 14 同步能力的扩展构建；
- 扩展已经加载或重新加载；
- 扩展加载后已经刷新宠物工坊标签页；
- 工坊地址属于可信 Studio 页面。

开发环境支持：

```text
http://localhost:<port>/studio
http://127.0.0.1:<port>/studio
```

项目部署支持主机名包含 `yk-pets` 的 `/studio` 路径，以及：

```text
https://yokry-he.github.io/yk-pets/studio
```

普通网站不能向扩展写入宠物配方。

### 14.2 同步步骤

1. 在宠物工坊修改一个容易识别的外观，例如主毛色、肚皮样式或胸口文字。
2. 点击“保存外观”。
3. Studio 先完成本地保存。
4. Studio 将外观包装为版本化配方信封。
5. 页面同步桥把配方发送给扩展。
6. 扩展验证配方并写入 `chrome.storage.local`。
7. 扩展返回成功或错误确认。
8. 已经运行的正式云灵读取新配方并更新外观。

### 14.3 扩展未安装时

本地 Studio 保存仍然成功。同步超时或没有检测到扩展，不会撤销本地配方。

### 14.4 如何验证同步

建议至少验证：

- 主毛色变化；
- 五种肚皮中的一种；
- 胸口显示模式；
- 胸口字母大小或位置；
- 后背 `YK` 位置；
- 正式网页宠物动作仍然可以播放。

### 14.5 同步后没有立即更新

1. 确认工坊点击过“保存外观”，不是只停留在未保存状态。
2. 在扩展管理页重新加载扩展。
3. 刷新工坊页面，再保存一次。
4. 刷新普通网页，使其获得最新 Content Script。
5. 检查工坊 URL 是否属于可信路径。
6. 确认扩展存储中存在活动配方。

## 17. 权限、安全和隐私边界

### 17.1 扩展网页权限

当前开发版使用广泛 HTTP/HTTPS 页面权限，目的是让解压加载版本无需逐站配置即可进行审计。正式商店版计划收紧为用户触发和按站点授权。

### 17.2 网页是不可信输入

网页 DOM、属性、请求内容和审计证据都被视为不可信输入。扩展不会执行网页提供的 JavaScript，也不使用 `eval` 或 `new Function`。

### 17.3 Shadow DOM 隔离

网页内宠物和提示界面使用 Shadow DOM，减少宿主网页 CSS 对扩展界面的污染。

### 17.4 Studio 同步安全边界

只有可信 Studio 主机和路径可以发送配方。消息必须：

- 来自当前窗口；
- 与页面同源；
- 满足同步请求结构；
- 包含可规范化的配方信封。

### 17.5 Agent 安全

- 只监听 `127.0.0.1`；
- 使用随机口令认证；
- 只能访问指定项目根目录；
- 写入前检查路径和文件哈希；
- 需要用户明确确认；
- 只允许运行预定义检查脚本；
- 不接受网页传入任意 Shell 命令。

### 17.6 数据存储

当前主要数据保存在本机浏览器存储和项目目录中。不要在配方文字、Mock 响应或项目日志中保存不必要的敏感信息。

## 18. 无障碍、性能和减少动态效果

- 菜单按钮支持键盘 Tab 和 Enter；
- 提示卡使用状态区域反馈；
- 高亮层不接收鼠标事件；
- 窄屏菜单不应造成页面级横向滚动；
- 浏览器启用 `prefers-reduced-motion: reduce` 时，高能视觉效果和闲时动作会减少；
- 页面隐藏时会暂停闲时动作调度；
- Studio 不使用全屏 Bloom，以降低注入普通网页后的渲染成本；
- 低性能设备上出现卡顿时，可关闭轨道、降低粒子、避免烟花等高能动作。

## 19. 常见故障排查

### 19.1 网页右下角没有云灵

检查：

- 当前是否为普通 HTTP/HTTPS 页面；
- 扩展是否启用；
- 扩展是否刚更新但页面未刷新；
- Content Script 是否在控制台报错；
- 页面 CSP 或浏览器是否阻止扩展注入。

### 19.2 只出现字母 N 加载按钮

说明 3D Vue/TresJS 组件尚未成功加载。点击按钮仍可打开报告。请检查扩展构建、控制台错误和依赖是否完整。

### 19.3 `<yk-pet>` 显示没有可用渲染器

说明 Web Component 已注册，但 `extension-cloud-fox` 渲染适配器没有成功注册。重新构建扩展，检查 Vue 适配器模块和生产画布是否被打包。

### 19.4 Side Panel 无法连接当前页面

扩展重载后刷新目标标签页。旧标签页中的 Content Script 不会自动更新。

### 19.5 双击宠物没有开始审计

- 确认没有拖拽操作；
- 确认至少启用一条审计规则；
- 打开 Side Panel 查看页面错误；
- 确认页面不是浏览器受限页面。

### 19.6 定位框位置不正确

页面布局可能刚发生变化。重新选择问题或重新审计；对于持续动画页面，滚动后再执行一次定位。

### 19.7 预览无法应用

该问题可能没有可逆预览，或目标元素已经变化、删除或无法重新定位。重新运行审计。

### 19.8 Network Lab 没有请求

- 刷新页面重新触发请求；
- 确认是 Fetch/XHR；
- 扩展重载后刷新标签页；
- 确认站点网络能力已开启；
- 检查请求是否发生在 Service Worker、WebSocket 或扩展未覆盖的通道。

### 19.9 Mock 没有命中

检查 URL、方法、启用状态、站点开关和规则优先级。保存规则后重新触发请求，不要只刷新 Network Lab 面板。

### 19.10 Agent 无法连接

- Agent 进程是否仍在运行；
- 地址是否为 `ws://127.0.0.1:<port>`；
- 口令是否完整；
- Side Panel 是否为最新扩展版本；
- 项目根目录是否有效；
- 防火墙或安全软件是否拦截本机端口。

### 19.11 只能看到源码候选，没有确认写入

说明映射置信度不足。可以：

- 为元素添加稳定 ID 或 `data-testid`；
- 保留有意义的资源文件名；
- 手动检查候选文件；
- 不要强行绕过置信度限制。

### 19.12 写入被拒绝，提示哈希变化

目标文件在生成补丁后被其他工具修改。重新生成补丁，不要覆盖并发修改。

### 19.13 回滚被拒绝

补丁写入后文件又发生变化。请先保存后续修改，再手动通过 Git 或备份处理，不要强制回滚。

### 19.14 动作没有声音

查看[动作语音和音色](#12-动作语音和音色)，并确认页面已发生用户点击。

### 19.15 Studio 保存后扩展外观没有变化

查看[同步后没有立即更新](#145-同步后没有立即更新)。最常见原因是扩展重载后没有刷新 Studio 和普通网页标签页。

### 19.16 胸口标志看不到

- 检查胸口显示模式；
- 检查“显示”开关；
- 调整大小；
- 将前后位置向外移动；
- 检查是否被能量核心或肚皮遮挡。

### 19.17 后背标志被尾巴遮挡

切换背面视角，继续上移标志或向身体外侧移动；再播放尾巴相关动作确认动态遮挡。

### 19.18 Studio 导入 JSON 失败

确认文件是有效 JSON。旧配方可以缺少新字段，但顶层结构必须仍是外观对象，不能导入完整日志、数组或压缩文件。

### 术语

| 术语 | 含义 |
|---|---|
| YK-PETS | 产品品牌 |
| 云灵 / Zeph | 当前默认宠物名字 |
| 云狐 / Cloud Fox | 当前默认宠物物种 |
| Side Panel | 浏览器右侧的完整操作工作区 |
| Local Agent | 只在本机运行的项目源码协作服务 |
| Studio / 宠物工坊 | 外观配方编辑和动作验证页面 |
| 配方 | 描述外观和渲染选择的数据，不是完整 Three.js 场景 |
| `<yk-pet>` | 负责宠物渲染生命周期和适配器选择的 Web Component |
| `pet-core` | 框架无关的配方、物种、渲染器和同步协议包 |
