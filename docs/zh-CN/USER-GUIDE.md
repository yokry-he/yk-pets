# NOVA Browser Agent 使用操作手册

> 适用版本：v0.5.2 及以上  
> 文档目标：让第一次使用 NOVA 的用户能够独立完成安装、宠物交互、页面审计、网络分析、Mock 规则配置和 Local Agent 操作。

## 1. 产品组成

NOVA Browser Agent 由三部分组成：

| 部分 | 作用 | 是否必须 |
|---|---|---|
| Chrome 扩展 | 网页宠物、页面审计、网络实验室、Side Panel | 必须 |
| Local Agent | 读取本地源码、生成补丁、运行校验、回滚 | 仅源码修改时需要 |
| Playground | 官方演示与回归测试页面 | 仅开发测试时需要 |

日常使用网络分析与 Mock 时，只安装 Chrome 扩展即可。

## 2. 安装扩展

### 2.1 使用已编译压缩包

1. 解压 `nova-browser-agent-<version>-chrome.zip`。
2. 在 Chrome 地址栏输入 `chrome://extensions`。
3. 开启右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择刚刚解压后的目录；该目录顶层必须直接包含 `manifest.json`。
6. 固定 NOVA 图标，方便打开 Side Panel。
7. 刷新需要使用 NOVA 的普通 HTTP/HTTPS 页面。

不要直接选择 ZIP 文件，也不要选择包含扩展目录的上一级目录。

### 2.2 从源码构建后安装

```bash
corepack enable
pnpm install --frozen-lockfile
pnpm build:extension
```

然后加载：

```text
apps/extension/.output/chrome-mv3
```

## 3. 页面中的 3D 宠物

### 3.1 基本交互

| 操作 | 结果 |
|---|---|
| 单击宠物 | 打开或收起模式与环形功能菜单 |
| 双击宠物 | 启动当前页面审计 |
| 悬停宠物 | 播放轻量问候，并跟随鼠标视线 |
| 拖拽宠物 | 调整页面中的停靠位置 |
| 按 Esc | 收起当前菜单 |
| 点击菜单外区域 | 收起菜单 |

宠物菜单分为“功能、动作、工程”三种模式。模式按钮和下方功能按钮统一使用紫色视觉体系。

### 3.2 功能模式

功能模式用于页面审计和问题浏览：

1. 开始页面审计；
2. 查看上一个或下一个问题；
3. 定位并高亮问题元素；
4. 预览临时修复；
5. 打开完整报告。

临时修复只修改当前页面 DOM，不写入本地源码；刷新页面后恢复。

### 3.3 动作模式

动作超过 6 个时会分页。分页指示点固定在宠物阴影下方。

普通动作包括招手、跳跃、转圈、扑腾、玩耍、休息、伸懒腰、玩球和吃饭。高能动作包括：

- 空翻落地；
- 甩尾旋风；
- 飞扑接球；
- 能量爆发。

页面处于后台、菜单正在打开、Agent 忙碌或系统启用“减少动态效果”时，自动动作会暂停或降级。

### 3.4 工程模式

工程模式提供：

- 网络实验室；
- Local Agent 连接；
- 生成源码补丁；
- 确认写入；
- Typecheck/Test/Build；
- 回滚最近补丁。

复杂内容统一在右侧 Side Panel 中处理。

## 4. 打开 Side Panel

可通过以下方式打开：

- 点击浏览器工具栏中的 NOVA 图标；
- 从宠物“工程”模式进入网络实验室；
- 从宠物的审计或补丁操作进入详情。

Side Panel 与当前活动标签页绑定。切换标签页后，请确认顶部显示的网站域名正确。

## 5. 网络实验室快速上手

### 5.1 一键总开关

网络实验室顶部的总开关按网站 Origin 保存，例如：

```text
https://app.example.com
```

开启后允许执行：

- Fetch/XHR Mock；
- 真实 JSON 响应修改；
- 人为延迟。

关闭后立即停止上述行为。性能采集可以继续运行，便于观察真实请求。

### 5.2 请求列表

1. 打开“请求”页。
2. 刷新目标网页或重新执行业务操作。
3. 使用搜索框筛选 URL、方法或状态码。
4. 使用“只看慢请求”过滤超过当前阈值的请求。
5. 点击一条请求查看详情。

请求详情包含：

- URL 与 HTTP 方法；
- 状态码；
- 总耗时；
- 传输体积；
- DNS、连接、TLS、TTFB 和下载阶段；
- Mock、修改和延迟标记；
- 脱敏后的响应预览。

## 6. 创建 Mock 规则

### 6.1 从已存在的请求生成

1. 在“请求”页打开请求详情。
2. 点击“生成 Mock”。
3. 系统进入 Side Panel 全页规则编辑器。
4. URL、方法、状态码和响应预览会自动填入。
5. 修改规则名称和响应数据。
6. 点击“测试规则”，确认测试 URL 可以命中。
7. 点击“保存并启用”。
8. 重新执行网页操作，确认请求显示“Mock”标记。

完整 Mock 命中后不会调用真实接口。

### 6.2 Mock 尚不存在的接口

1. 打开“规则”页。
2. 点击“新增规则”。
3. 输入页面将要调用的接口路径，例如：

```text
/api/future-feature
```

4. 选择 HTTP 方法。
5. 选择 Glob 或正则匹配方式。
6. 配置状态码、响应类型、响应正文和延迟。
7. 使用测试 URL 验证规则。
8. 保存并启用。
9. 页面以后发起匹配的 Fetch/XHR 时，NOVA 直接返回自定义响应。

服务器上不需要真实存在该接口。

### 6.3 复制规则

复制适合快速制作多种业务状态：

- 成功；
- 空数据；
- 401 登录过期；
- 403 权限不足；
- 500 服务异常；
- 高延迟。

复制后的规则拥有新 ID，默认关闭。修改名称、匹配条件或返回数据后再启用，避免和原规则冲突。

## 7. 规则匹配说明

规则按照以下顺序决定谁先执行：

1. 优先级更高；
2. 匹配更精确；
3. 更新时间更新。

### 7.1 Glob 示例

```text
/api/users/*
https://api.example.com/v1/*
/api/orders?status=*
```

### 7.2 正则示例

```text
^https://api\.example\.com/v2/users/\d+$
```

错误的正则不会命中请求。保存前应使用“测试规则”。

### 7.3 Query 条件

支持：

- 等于；
- 包含；
- 参数存在。

例如：

```text
URL: /api/orders
Query: status equals pending
```

## 8. Mock 响应编辑

支持 JSON 和文本。

### 8.1 JSON 示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1001,
    "name": "NOVA"
  }
}
```

JSON 专注编辑模式提供格式化和语法校验。无效 JSON 不能作为 JSON 类型保存。

### 8.2 状态码与延迟

可配置：

- 200/201 等成功响应；
- 400/401/403/404；
- 429；
- 500/502/503；
- 固定延迟。

延迟只用于开发测试，不应在不知情的情况下长期保持开启。

## 9. 修改真实接口响应

“修改响应”会先访问真实接口，再对 JSON 结果执行转换。

支持：

- `set`：设置或替换字段；
- `remove`：删除字段。

示例：

```text
set data.user.vip = true
remove data.internalToken
```

路径可指向嵌套对象或数组。非 JSON 响应不会执行 JSON 修改。

## 10. 性能分析与图表

“概览”页提供：

- 请求总数；
- 慢请求数；
- 错误数；
- Mock 数量；
- 总传输体积；
- 平均耗时；
- P95；
- 综合评分。

主要图表：

1. 最慢请求排行；
2. 资源类型与传输体积分布；
3. 最近请求瀑布图。

常见诊断：

- TTFB 过高：优先检查服务端处理、数据库和下游依赖；
- 下载时间过高：检查体积、压缩和网络；
- 图片或脚本过大：检查格式、拆分和按需加载；
- 重复请求：检查缓存、去重和组件生命周期；
- Mock 后仍然慢：瓶颈可能在前端解析或渲染。

## 11. Local Agent 操作

### 11.1 启动

```bash
pnpm dev:agent
```

终端显示 WebSocket 地址和一次性连接口令。

### 11.2 连接

1. 打开 Side Panel 的 Agent 区域。
2. 输入地址，例如 `ws://127.0.0.1:4736`。
3. 输入终端显示的 Token。
4. 连接成功后再生成源码补丁。

### 11.3 安全流程

```text
发现问题
→ 生成候选补丁
→ 用户审查 Diff
→ 明确确认写入
→ 运行校验
→ 必要时回滚
```

NOVA 不会在没有用户确认时直接覆盖源码。

## 12. 数据与隐私

- Mock 配置保存在本机 Chrome Storage；
- 请求预览会限制长度、层级和字段数量；
- Token、Cookie、密码、Secret 等字段会脱敏；
- 当前版本不会上传网络记录；
- 规则默认限定创建时的网站 Origin；
- 使用完 Mock 后建议关闭网站总开关。

## 13. 推荐日常工作流

### 接口尚未开发

```text
手动新增 Mock
→ 前端联调
→ 复制出成功/空数据/异常版本
→ 后端完成后关闭 Mock
```

### 已有接口返回不满足测试条件

```text
捕获真实请求
→ 生成 Mock 或修改响应
→ 保存规则
→ 验证页面状态
```

### 分析慢页面

```text
打开性能采集
→ 刷新页面
→ 查看最慢请求和瀑布图
→ Mock 慢接口再次加载
→ 判断网络端还是前端端瓶颈
```

## v0.6.0 宠物表演、生活状态与彩蛋

动作菜单新增：害羞偷看、星星杂耍、云朵午睡、星光喷嚏和高空烟花秀。动作超过六项时使用宠物阴影下方的分页点切换页面，也可以在菜单区域滚动鼠标滚轮。

### 高空烟花秀

1. 打开宠物菜单；
2. 切换到“动作”；
3. 翻页找到“高空烟花秀”；
4. 点击后保持当前标签页可见。

烟花会连续发射三次，每次随机选择精选花型和色板。烟花期间宠物会抬头观看，触角、尾尖和胸口能量核心同步增强。

### 自动彩蛋

星光喷嚏和高空烟花可能在长时间闲置后低概率触发。自动彩蛋不会在以下状态播放：

- 菜单打开；
- Agent 或页面审计忙碌；
- 标签页处于后台；
- 系统开启减少动态效果。

用户可随时通过动作菜单主动触发，不必等待随机彩蛋。

## v0.6.1 动画连续性与主动发光操作

### 连续触发动作

当一个动作正在播放时，再点击另一个动作不会中断当前动作。NOVA 会记住下一动作，并在当前动作完成、经过短暂稳定阶段后继续播放。队列只保留一个下一动作，连续快速点击时优先保留优先级更高或最后一次同级操作。

### 鼠标悬停

鼠标移入只会在 NOVA 完全待机时播放一次招手。烟花、玩球、午睡、空翻等动作播放期间，移入或移出都不会改变当前动作。鼠标移出仍会让视线逐渐回到中间。

### 主动发光动作

在“动作”模式中翻页后可找到：

- **触角聚能**：触角向中心靠拢，触角尖端、胸口核心和数据环同步高亮；
- **尾光流动**：尾巴中段、尾尖、拖尾和星点形成连续能量流。

### 云朵午睡

“云朵午睡”会让 NOVA 侧躺在云朵上。午睡期间眼睛闭合、四肢收拢、尾巴低频摆动，并显示缓慢上升的睡眠符号。
