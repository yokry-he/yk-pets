# 按需提示与动作运行时设计

版本：`0.2.6`

## 目标

v0.2.6 解决两个问题：

1. 大型状态气泡不应在功能菜单打开期间持续占据页面。
2. 动作按钮必须可靠地重新触发 3D 动画，即使用户连续点击同一个动作。

## 按需提示

状态气泡默认隐藏，只在以下事件发生时显示：

- 页面审计开始或完成；
- 页面定位、预览和撤销返回结果；
- Local Agent 开始或完成补丁、验证和回滚任务；
- 发生错误或需要用户处理的重要状态。

一般结果显示约 4.8～5.2 秒，错误显示约 8 秒。运行中的任务保持显示，但用户仍然可以手动关闭。

提示卡只保留两行摘要和当前问题名称。存在详细内容时，卡片显示“在侧边栏查看详情”，将用户带到 Side Panel。完整问题列表、Diff、命令输出和错误详情不在网页悬浮层展开。

## 菜单与提示分层

提示卡、菜单切换器和环形按钮属于不同层级：

- 没有提示时，功能／动作切换器位于悬浮区域顶部；
- 提示出现时，切换器和菜单关闭按钮自动向下避让；
- 提示关闭后，控制区恢复原位；
- 提示卡可以覆盖宿主页面，但不会永久占位。

## 动作重新触发

仅改变 `behavior` 无法保证连续播放同一个动作，因为第二次点击时状态值可能没有变化。

v0.2.6 增加 `motionKey`：

```text
动作按钮点击
  → motionNonce + 1
  → AvatarCanvas.motionKey
  → CloudFox.motionKey
  → 重置 behaviorStartedAt
```

因此连续点击“跳跃”或“转圈”时，动画时间轴会从零重新开始。

## 动作反馈

手动动作不再打开大型提示卡，而是在宠物附近显示约 1.5 秒的小型动作标签，例如：

```text
跳跃 · Jump
```

闲时动作不显示标签，避免打扰用户。

## 状态优先级

```text
审计或 Agent 忙碌状态
  > 手动动作
  > 闲时轮播
  > 默认页面健康动作
```

忙碌状态开始时会停止手动动作和闲时轮播。任务结束后才恢复轮播计时。
