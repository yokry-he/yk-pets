# 补丁计划 v0.7.6

`@yk-pets/pet-patch-plan` 只描述修改，不直接访问工作区。

## 安全约束

- 路径必须是项目相对路径，并统一使用 `/`。
- 拒绝绝对路径、盘符、`..`、`.`、控制字符和反斜杠。
- 默认保护 `.git/`、`.yk-pets/approvals/` 与 `node_modules/`。
- 同一路径不能在一个计划中被多个 Change 重复触及。
- `update`、`delete`、`move` 必须包含小写 SHA-256 原文件摘要。
- 更新编辑必须按偏移升序排列且不能重叠；建议总是提供 `expectedText`。

计划摘要由稳定 JSON 序列化后计算 SHA-256，审批令牌绑定该摘要，因此审批后对计划的任何修改都会失效。
