# 受控仓库发布 v0.7.7

`RepositoryPublisher` 将发布过程固定为：消费一次性审批 → 精确暂存审批路径 → 读取真实 Git 快照 → 执行提交门禁 → 创建提交 → 可选推送 → 可选创建 Draft PR → 无论完整成功或提交后部分失败都追加账本。

`yk-pets.publish-approval/v1` 绑定用户、项目、仓库、会话、计划与摘要、基线、分支、提交标题、动作集合、远端、PR 仓库与目标分支。令牌最多存活 10 分钟且只能消费一次。

推送或 PR 在提交后失败时结果为 `partial`，不会假装仓库没有提交；错误会写入账本，调用方应展示恢复指引或继续发布入口。
