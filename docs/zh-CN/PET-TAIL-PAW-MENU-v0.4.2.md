# NOVA 尾巴、前爪动作与菜单配色设计 v0.4.2

## 目标

本版本针对实际 Chrome 页面中的三个视觉问题进行修正：尾巴向身体内侧回卷、前爪根部随动作大幅漂移，以及功能／动作／工程模式下方按钮配色不一致。

所有修改直接进入 Vue/TresJS 与菜单 CSS 源码，并在清理旧 `.output`、`.wxt` 后通过 WXT 重新构建。

## 尾巴外展

- 尾根放置于身体左后侧：`(-0.58, -0.48, -0.34)`。
- 尾根、中段、尾尖三条曲线都沿负 X 方向继续延伸，尾尖不再折返身体中心。
- 尾根摆幅降低，只负责传递方向；中段与尾尖逐级增加摆幅和相位延迟。
- 尾尖辉光、拖尾和星点同步移动到新的外展末端。
- 伸懒腰时尾巴整体轻微上扬，但仍保持在身体外侧。

## 前爪三级控制

每只前爪拆分为：

```text
ShoulderRoot
└── Forearm
    └── PawTip
        └── PropAnchor
```

- `ShoulderRoot` 固定在胸口两侧，只允许轻微位置偏移和小角度配合。
- `Forearm` 负责抬起、前伸、收拢与主要摆幅。
- `PawTip` 负责挥动、拍击和触碰后的回弹。
- 道具锚点继续位于爪端，因此球仍能随新的关节层级正确生成。

## 动作调整

- 待机：肩部保持稳定，仅随呼吸产生极小浮动。
- 招手：肩部轻抬，前臂抬到脸侧，左右挥动集中在爪端。
- 跳跃：肩部仅轻微内收，前臂和爪端向胸前折叠。
- 玩球：肩部只做小幅前送，拍击由前臂与爪端完成，并保留接触回弹。
- 吃饭：肩部稳定，前爪自然支撑在饭盆两侧，主要动作来自头部和嘴部。
- 伸懒腰：肩带跟随胸部小幅前移，前臂充分向前下方伸展，爪端承担落地支撑。
- 扑腾与玩耍：肩部只做轻微配合，主要摆幅转移到前臂和爪端。

## 统一菜单配色

功能、动作、工程三种模式的六槽位按钮统一使用当前功能模式的紫色体系：

```css
--pet-menu-button-border
--pet-menu-button-icon
--pet-menu-button-icon-bg
--pet-menu-button-icon-border
--pet-menu-button-glow
--pet-menu-button-hover-border
--pet-menu-button-hover-glow
```

已删除动作模式的青绿色覆盖和工程模式的黄色覆盖。危险工程操作仍通过 Tooltip 和确认流程表达风险，不再改变按钮基础配色。
