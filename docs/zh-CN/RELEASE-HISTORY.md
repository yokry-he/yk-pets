# NOVA 版本与验证历史

本文合并原先散落在仓库根目录和 `docs/zh-CN` 中的逐版本 Release Notes、专项设计说明与验证报告。当前文档只保留对维护和发布仍有价值的结论；需要查看原始过程记录时，可从 Git 标签 `v0.6.10` 或 Git 历史恢复。

## 当前稳定基线

- 当前扩展版本：`v0.6.10`；
- 交付形态：Chrome/Edge Manifest V3 扩展；
- 核心技术：WXT、Vue 3、Three.js、TresJS、TypeScript；
- 本地协作：Token 认证的 WebSocket Agent；
- 构建目录：`apps/extension/.output/chrome-mv3`；
- 商店上传包：`apps/extension/.output/novaextension-0.6.10-chrome.zip`。

## 当前验证命令

在仓库根目录执行：

```bash
corepack enable
pnpm install --frozen-lockfile
pnpm check:documentation
pnpm typecheck
pnpm test
pnpm test:network-domain
pnpm test:network-workbench
pnpm build:agent
pnpm build:playground
pnpm build:extension
pnpm zip:extension
```

任一质量门禁失败时，不应继续发布。

## 必须保留的手动回归

自动化不能完全替代以下浏览器检查：

- 在浅色和深色网页上检查宠物、尾尖、菜单和拖拽；
- 检查动作打断、闲时动作、页面隐藏和减少动画偏好；
- 检查页面审计范围、问题定位和可撤销预览；
- 检查 Fetch/XHR Mock、延迟和完整 JSON 修改；
- 检查 Local Agent 的连接、确认、验证和回滚；
- 在当前系统上试听“萌系少女”和“萌宠伙伴”，记录 Chrome 实际选择的中文 TTS 声线。

系统萌系音色依赖设备已安装的语音，自动化只能验证调用链和参数，不能保证所有操作系统上的最终听感一致。

## 后续记录规则

- 不再在仓库根目录创建 `RELEASE-NOTES-vX.Y.Z.md` 或 `VALIDATION-vX.Y.Z.*.md`；
- 每个版本只更新本文、项目状态、Manifest/包版本和必要的操作文档；
- Chrome ZIP、源码归档、SHA-256 和面向用户的发布说明保存在 GitHub Release 或发布系统中；
- 版本专属自动化脚本只有在仍属于回归门禁时才保留，不能只作为一次性升级记录存在。
