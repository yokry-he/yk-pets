# NOVA 宠物思考气泡与审计操作（v0.3.0）

## 设计目标

网页内不再长期展示与当前宠物视觉不一致的大矩形报告卡。状态与审计结果改为从宠物头部附近延伸出的思考气泡，并置于悬浮交互的最上层。

## 思考气泡

- 默认不显示，只在审计、预览、错误或 Agent 任务需要反馈时出现。
- 气泡固定在宠物头部左上方，通过两个渐缩圆点与宠物头部形成视觉连接。
- 气泡可以覆盖环形按钮和功能切换器，避免为了避让而把信息推得过远。
- 气泡自身提供关闭按钮，用户可以随时关闭。
- 简短结果保留在网页内；完整问题列表、Diff 和检查输出进入浏览器右侧栏。

## 审计结果操作

审计完成并发现问题后，以下操作从宠物环形菜单移动到思考气泡：

- 上一个问题
- 下一个问题
- 预览修复 / 撤销预览
- 完整报告

网页内的“功能”环只保留：

- 页面审计 / 重新审计
- 工程工具

这样可以减少常驻按钮数量，并让问题浏览操作与审计结果保持在同一上下文中。

## 右侧栏打开链路

旧实现会先异步写入待处理动作，再请求打开 Side Panel。Chrome 对 Side Panel 的打开要求保留用户手势，前置异步操作可能导致打开请求失效。

新流程在收到点击消息后立即调用 `chrome.sidePanel.open()`，待处理动作的存储与面板打开并行执行。打开失败时，宠物气泡会显示明确错误，不再静默无响应。
