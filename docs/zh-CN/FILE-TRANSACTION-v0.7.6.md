# 文件事务与回滚 v0.7.6

事务分为预检、顺序提交和逆序恢复：

1. 读取全部触及路径，拒绝目录和符号链接。
2. 验证项目 ID、基线版本、缺失/存在状态和 SHA-256。
3. 计算更新后的完整内容及实际写入字节。
4. 每个写入或删除使用 CAS 前置条件。
5. 任一操作失败后，使用已完成操作的 Journal 逆序恢复。

回滚也使用 CAS。若事务后文件又被外部修改，SDK 返回 `rollback-failed` 并保留恢复日志，不会覆盖外部内容。

`FileTransactionResult.journal` 可能含完整源码，只能存放于可信内存或加密存储。外部展示请使用 `toPublicTransactionSummary()`。
