# 网页内 3D 云灵（Zeph）交互设计

## 1. 身份

```text
产品：YK-PETS
宠物名字：云灵（Zeph）
宠物物种：云狐（Cloud Fox）
```

本文描述的是云灵作为 YK-PETS 当前默认宠物的交互方式。产品品牌与宠物身份不得混用。

## 2. 设计目标

YK-PETS 的主要入口位于允许扩展运行的网页右下角。用户无需先理解浏览器扩展结构，只需与云灵互动，即可触发页面审计、问题定位、临时预览、本地 Agent、源码补丁、验证与回滚。

Side Panel 继续承担详细报告、连接凭据、Network Lab、Diff 和命令输出，但主要流程都可以由云灵发起。

## 3. 默认位置与显示

- 云灵固定在网页右下角，默认距离右侧和底部各 `18px`；
- 使用 Shadow DOM 避免宿主网页 CSS 污染；
- 3D 模型在浏览器空闲阶段加载；
- Canvas 使用透明背景和受限 DPR；
- 用户可以拖拽，位置限制在当前视口；
- 页面包含问题时显示问题数量徽标；
- 审计、补丁或验证运行期间显示动作与可读状态；
- 宠物介绍使用“云灵（Zeph）· 云狐（Cloud Fox）”。

## 4. 动物交互语义

| 动物交互 | 结果 |
|---|---|
| 单击云灵 | 展开或收起快捷功能菜单 |
| 双击云灵 | 立即执行页面审计 |
| 右键云灵 | 打开本地工程工具箱 |
| 悬停云灵 | 打招呼并跟随指针 |
| 拖拽云灵 | 调整网页内位置 |
| 点击问题数量 | 进入问题导航 |

所有交互同时提供键盘可访问按钮和可读名称。双击与右键不是唯一入口。

## 5. 快捷功能菜单

功能菜单包括：

1. **页面审计**：读取 DOM、资源、性能和无障碍信息；
2. **完整报告**：在 Side Panel 查看结果和指标；
3. **问题导航**：上一条、下一条、定位当前问题；
4. **预览修复 / 撤销预览**：应用安全的临时 DOM 修改；
5. **动作**：控制云灵的动作与表现；
6. **工程工具**：连接 Local Agent、生成补丁、验证和回滚；
7. **网络实验室**：查看请求并配置 Mock、延迟和响应修改。

当尚未审计或条件不足时，操作保持禁用并说明原因。

## 6. 本地工程工具箱

工程动作包括：

1. **连接 YK-PETS Local Agent**；
2. **生成源码补丁**；
3. **确认写入补丁**；
4. **运行项目检查**；
5. **回滚最近补丁**。

网页内宠物只发送受约束动作，不接收或执行任意 Shell 命令。

## 7. 状态反馈

| 操作状态 | 云灵行为 | 示例文案 |
|---|---|---|
| 等待互动 | idle / listening | 摸摸我可以打开所有功能。 |
| 页面审计 | thinking | 我正在检查页面结构、资源和无障碍状态。 |
| 页面健康 | happy | 页面状态很好。 |
| 高优先级问题 | confused | 我发现了需要先处理的问题。 |
| 问题定位 | listening | 这是第 2 个问题：图片没有声明固有尺寸。 |
| 临时预览 | happy | 临时修改已经应用。 |
| 连接 Agent | thinking | 正在连接 YK-PETS Local Agent。 |
| 补丁可写入 | happy | 源码补丁已经准备好。 |
| 补丁写入成功 | excited | 补丁已经写入项目。 |
| 验证全部通过 | excited | 所有项目检查都通过了。 |
| 失败或条件不足 | confused | 显示明确原因和下一步。 |

## 8. 页面与 Side Panel 协作

```text
网页内云灵（Zeph）
├─ 本地完成：审计、问题切换、定位、预览、撤销
└─ 委托 Side Panel：连接 Agent、生成补丁、写入、验证、回滚
```

委托流程：

1. Content Script 将动作和当前问题 ID 发送给 Background；
2. Background 持久化待执行动作并打开 Side Panel；
3. Side Panel 消费动作并执行；
4. Side Panel 将状态同步回 Content Script；
5. 云灵根据状态切换动作、文案和忙碌指示。

## 9. 品牌兼容

`v0.6.10` 旧组件仍可能产生 NOVA 技术文案。`apps/extension/brand.ts` 会在普通 DOM 和开放 Shadow Root 中统一转换：

- 产品名称转换为 YK-PETS；
- 宠物相关名称转换为云灵（Zeph）；
- Storage Key 在 `nova:*` 和 `yk-pets:*` 之间迁移和镜像。

后续逐组件完成原生重命名后，应删除这一过渡层。

## 10. 响应式、性能和无障碍

- 桌面端菜单显示在云灵左侧，窄屏时堆叠到上方；
- 菜单宽度受视口限制；
- 只有宠物和菜单区域响应指针；
- 标签页隐藏时应暂停渲染；
- 低性能设备使用较低 DPR 或 2D 回退；
- `prefers-reduced-motion` 下降低动作幅度；
- 所有按钮提供可访问名称。
