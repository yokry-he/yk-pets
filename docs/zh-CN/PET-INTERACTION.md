# 网页内 3D NOVA 交互设计

## 1. 设计目标

NOVA 的主要入口从 Side Panel 转移到允许扩展运行的网页右下角。用户不需要先理解浏览器扩展结构，只需与 3D 云狐互动，即可触发页面审计、问题定位、临时预览、本地 Agent、源码补丁、验证与回滚。

Side Panel 继续承担详细报告、连接凭据、Diff 和命令输出，但所有主要流程都可以由云狐发起。

## 2. 默认位置与显示方式

- 云狐固定在网页右下角，默认距离右侧和底部各 `18px`。
- 容器使用 Shadow DOM，避免宿主网页 CSS 污染。
- 云狐加载安排在浏览器空闲阶段，不阻塞页面首屏逻辑。
- 3D Canvas 使用透明背景，不显示额外面板底色。
- 用户可以拖拽云狐，位置被限制在当前视口内。
- 页面包含问题时，云狐右上方显示问题数量徽标。
- 审计、补丁或验证运行期间，显示忙碌状态和思考动作。

## 3. 动物交互语义

| 动物交互 | 结果 |
|---|---|
| 单击云狐 | 展开或收起快捷功能菜单 |
| 双击云狐 | 立即执行页面审计 |
| 右键云狐 | 打开本地工程工具箱 |
| 悬停云狐 | 云狐打招呼并跟随指针 |
| 拖拽云狐 | 调整网页内位置 |
| 点击问题数量 | 通过菜单进入问题导航 |

所有交互同时提供键盘可访问按钮和可读名称。双击与右键是快捷方式，不是唯一入口。

## 4. 快捷功能菜单

单击云狐后显示六个主要操作：

1. **页面审计**：读取 DOM、资源、性能与无障碍信息，生成 `AuditReport`。
2. **上一个问题**：切换到上一条审计问题并在网页中定位。
3. **下一个问题**：切换到下一条审计问题并在网页中定位。
4. **预览修复 / 撤销预览**：对当前问题应用安全的临时 DOM 修改，或恢复原始状态。
5. **详细报告**：打开 Side Panel 并滚动到报告区域。
6. **工程工具**：展开 Local Agent 操作。

当尚未审计或当前问题不支持预览时，相关操作保持禁用，并由云狐说明原因。

## 5. 本地工程工具箱

工程工具通过云狐右键或“工程工具”按钮打开：

1. **连接本地 Agent**：打开 Side Panel 的 WebSocket 地址和 Token 区域；已有凭据时自动尝试连接。
2. **生成源码补丁**：将当前问题和页面地址发送给 Local Agent，搜索源码并生成最小 Diff。
3. **确认写入补丁**：只有存在可应用补丁时执行；Local Agent 在写入前验证哈希并创建备份。
4. **运行项目检查**：执行允许列表中的 `typecheck`、`test`、`build`。
5. **回滚最近补丁**：当前文件仍与补丁后哈希一致时恢复备份。

浏览器中的动物只发出受约束动作，不接收或执行任意 Shell 命令。

## 6. 状态反馈

| 操作状态 | 云狐行为 | 示例文案 |
|---|---|---|
| 等待互动 | idle / listening | 摸摸我可以打开所有功能。 |
| 页面审计 | thinking | 正在检查页面结构、资源和无障碍状态。 |
| 页面健康 | happy | 页面状态很好。 |
| 高优先级问题 | confused | 我发现了需要先处理的问题。 |
| 问题定位 | listening | 这是第 2 个问题：图片没有声明固有尺寸。 |
| 临时预览 | happy | 临时修改已经应用。 |
| 连接 Agent | thinking | 正在连接本地项目 Agent。 |
| 补丁可写入 | happy | 源码补丁已经准备好。 |
| 补丁写入成功 | excited | 补丁已经写入项目。 |
| 验证全部通过 | excited | 所有项目检查都通过了。 |
| 失败或条件不足 | confused | 显示明确原因和下一步。 |

## 7. 页面与 Side Panel 协作

```text
网页内 3D NOVA
  ├─ 本地完成：审计、问题切换、定位、预览、撤销
  └─ 委托 Side Panel：连接 Agent、生成补丁、写入、验证、回滚
```

委托流程：

1. Content Script 将待执行动作与当前问题 ID 发送给 Background Service Worker。
2. Service Worker 将动作写入 `chrome.storage.local`，再打开 Side Panel。
3. Side Panel 启动后消费待执行动作；若 Side Panel 已打开，则通过 Runtime Message 立即处理。
4. Side Panel 将执行状态发送回 Content Script。
5. 网页中的云狐根据状态切换动作、文案和忙碌指示。

待执行动作保存在存储中，避免 Service Worker 休眠或 Side Panel 初始化延迟导致命令丢失。

## 8. 响应式与避让

- 桌面端快捷菜单显示在云狐左侧，工程工具箱位于其上方。
- 窄屏设备将菜单堆叠到云狐上方。
- 所有菜单宽度受视口限制，长标题使用省略或换行。
- 高亮标签继续使用目标元素外部自动避让，不与云狐交互层混合。
- 云狐层使用最高扩展层级，但只有动物与菜单区域响应指针，避免覆盖页面大面积内容。

## 9. 性能预算

当前版本将 3D 组件延迟到浏览器空闲阶段初始化，并限制 DPR。后续优化目标：

- 将审计采集器与 3D 宠物拆成独立 Content Script bundle。
- 在低性能设备使用更低 DPR 或 2D 回退。
- 标签页隐藏时暂停渲染。
- 页面处于 `prefers-reduced-motion: reduce` 时降低动作幅度。
- 支持按站点关闭或恢复 3D 动物。
