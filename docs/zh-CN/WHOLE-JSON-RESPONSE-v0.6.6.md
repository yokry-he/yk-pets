# NOVA v0.6.6：真实响应完整 JSON 编辑

## 新交互

“修改真实响应”不再提供“设置字段 / 删除字段 / JSON 路径 / 字段值”表单，而是直接显示一份完整 JSON 编辑器。

从请求列表选择“基于请求生成规则”时，编辑器会优先带入该请求真实返回的完整 JSON。可以直接新增、删除或修改任意层级、数组和对象，保存后整份 JSON 会成为网页最终收到的响应正文。

## 运行方式

- 请求仍会发送到真实接口。
- 保留真实接口的 HTTP 状态码、状态文本和响应头。
- 仅将 JSON 正文替换为规则中保存的完整 JSON。
- Fetch 与 XHR 使用相同语义。
- 最多采集 512 KB 的可编辑 JSON，避免大型响应占用过多内存。

## 兼容性

v0.6.5 及更早版本创建的字段级响应规则仍能运行。打开并重新保存旧规则后，会迁移到完整 JSON 替换格式。

## 使用步骤

1. 刷新网页并重新触发目标请求。
2. 在请求详情中点击“基于请求生成规则”。
3. 选择“修改真实响应”。
4. 在“真实响应 JSON”中直接修改整份内容。
5. 保存并启用，然后再次触发请求。
