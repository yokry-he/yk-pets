# 验收、性能与安全 / Acceptance, Performance, and Safety

## 1. 自动检查层级

### 配置与迁移

- 默认配方可以创建；
- 旧配方自动补齐新增字段；
- 导入、导出后字段往返一致；
- 修改一个局部字段不会覆盖无关字段；
- 枚举、颜色和数值正确归一化；
- 协议信封版本和物种外观版本明确区分。

### Rig 静态检查

- 根、身体和头部存在；
- 左右眼与瞳孔存在并独立；
- 四肢挂点存在；
- 前爪包含肩、前臂、爪端和 Anchor；
- 尾巴能力开启时包含根、中、尖；
- 胸口、后背、嘴和道具 Anchor 存在；
- 场景特效有独立根节点。

### 动作检查

- 动作 ID 唯一；
- 注册表时长大于零或明确为持续状态；
- 手动动作可重复播放；
- 动作声明所需挂点；
- 道具动作使用共享运动姿态；
- 尾巴旋转不修改整身 Y 旋转；
- 爆炸粒子不存在回收公式；
- 退出阶段包含 settle。

### 渲染器检查

- `rendererId` 唯一；
- `supports(speciesId)` 正确；
- mount、update 和 destroy 完整；
- `<yk-pet>` 可以解析渲染器；
- 组件卸载时释放资源；
- WebGL 不可用时有明确错误或回退。

## 2. 四视角矩阵

每个主要动作至少检查：

```text
正面
左侧
背面
右侧
```

### 待机

- 呼吸不过度；
- 四肢稳定；
- 眼睛挂点正确；
- 尾巴连接自然；
- 胸背标志清晰；
- 地面阴影完整。

### 招手

- 肩部稳定；
- 前臂抬起；
- 爪端围绕手腕摆动；
- 另一只爪有轻微辅助；
- 结束平滑。

### 跳跃

- 有蓄力、上升、最高点、下落、落地和恢复；
- 后爪收起并落地缓冲；
- 身体不会突然瞬移；
- 阴影与高度关系合理。

### 玩球或接球

- 双眼全程看球；
- 球和前爪处于同一构图；
- 后爪参与；
- 球不出现在身体完全相反侧；
- 道具没有明显穿模。

### 吃饭

- 桌子、饭盆和食物关系正确；
- 嘴部对准饭盆；
- 前爪靠近桌沿；
- 桌子不穿身体或地面；
- 道具退出自然。

### 休息或睡眠

- 承托物完整；
- 身体在承托物上方；
- 四肢不穿底；
- 闭眼稳定；
- 退出不突然坠落。

### 尾巴动作

- 身体基本稳定；
- 尾根连接保持；
- 中段和尾尖有延迟；
- 不露出截面；
- 结束不反向倒转。

### 能量动作

- 充能球位于正确 Anchor；
- 双爪或触角姿态与球体一致；
- 爆炸足够明显；
- 粒子外散并淡出；
- 常驻轨道不被误当作冲击波。

## 3. 参考图对照

使用参考图时，检查以下视觉指标：

- 头身比；
- 身体宽高与厚度；
- 眼睛尺寸、间距和垂直位置；
- 耳朵或触角轮廓；
- 四肢嵌入点和长度；
- 尾根位置、尾巴方向和尾尖尺寸；
- 胸前和后背标志；
- 主色、阴影和发光色；
- 正面识别度。

参考图通常带有透视、姿势和镜头畸变。先重建稳定正交比例，再通过相机复现视觉效果。

## 4. 性能预算

适用于网页内常驻宠物的建议预算：

- 默认 DPR 上限 1.4，紧凑覆盖层可更低；
- 不为普通宠物启用全屏 Bloom；
- 粒子使用可控上限和低段数几何；
- 常驻 Torus、Tube 和高段数 Sphere 数量受控；
- 不在每帧创建新的纹理、材质或大量向量；
- 可复用临时 Vector3、Euler 和 Color；
- 页面隐藏时暂停闲时调度，并根据运行时能力暂停或降低渲染；
- 低性能模式关闭高能粒子和场景特效；
- `prefers-reduced-motion` 下禁用高能闲时动作。

## 5. 资源释放

卸载或切换渲染器时必须：

- 停止 `requestAnimationFrame`、Loop 回调和计时器；
- 移除 DOM、Window、Document 和 Chrome Storage 监听器；
- 释放 CanvasTexture；
- 释放动态 Geometry 和 Material；
- 停止音频和 TTS；
- 清空动作队列；
- 销毁 Vue 或其它框架实例。

## 6. 安全边界

- 配方视为不可信 JSON；
- 不执行配方中的函数或字符串代码；
- 不接受任意脚本 URL；
- 动作 ID 必须来自注册表；
- 物种和渲染器必须来自注册表；
- 导入文件限制大小和 JSON 类型；
- 外部 GLB 需要单独的来源、大小、许可和解析策略；
- Web Component 属性中的 JSON 必须捕获解析错误；
- Studio→扩展同步必须限制可信 Origin 和路径；
- 不让宠物动作直接访问 Shell、源码路径或任意 Chrome API。

## 7. 无障碍

- 3D Canvas 作为装饰时使用 `aria-hidden`；
- 可交互外壳使用按钮语义；
- 动作菜单支持键盘；
- 状态提示使用适当的 `aria-live`；
- 动作和语音可关闭；
- 减少动态效果下提供低运动版本；
- 颜色不能是唯一状态提示。

## 8. CI 完成条件

至少通过：

```bash
pnpm check:documentation
pnpm check:3d-pet-skill
pnpm typecheck
pnpm test
pnpm build
pnpm build:playground
```

若项目提供宠物专用检查，也必须运行。

## 9. 人工检查记录

最终报告必须明确区分：

- 自动检查已通过；
- 哪些动作已人工观察；
- 哪些视角已观察；
- 哪些设备或浏览器未验证；
- WebGL、声音和扩展同步是否实际测试。

连接器或无界面环境不能观察 WebGL 时，应明确写出限制，不得用“CI 通过”代替视觉验收。

## 10. 完成判定

- 宠物静态外形与参考目标一致；
- Rig 结构稳定；
- 四肢、尾巴、眼睛和标志可独立控制；
- 显式动作和闲时动作都可运行；
- 道具、追视和身体动作共享坐标；
- 配方持久化与迁移正确；
- 性能、安全和资源释放符合要求；
- 自动门禁通过；
- 人工视觉矩阵已执行或明确标记为待执行。
