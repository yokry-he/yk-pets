# NOVA 宠物前爪与道具对齐设计 v0.4.1

## 目标

本版本解决 v0.4.0 可加载包中模型结构与运行时动作不一致的问题，所有修改直接进入 Vue/TresJS 源码并通过 WXT 重新构建，不再向旧生产包叠加运行时脚本。

## 前爪基础姿态

- 左右前爪基础挂点调整到胸口两侧：`x = ±0.50`、`y = -0.12`、`z = 0.82`。
- 前爪略向身体前方突出，默认姿态与招手、玩球、吃饭、伸懒腰共用同一组基础坐标。
- 每只前爪增加独立锚点，后续道具不再依赖固定页面坐标。

## 玩球动作

- 总时长由 5.6 秒延长到 8.4 秒。
- 能量球从左前爪末端附近凝聚出现。
- 动作包含凝聚观察、第一次拍球、跨身体弹跳、另一侧回拍、回到两前爪中间并消散。
- 宠物头部视线根据球的实时位置转动。
- 球的位置基于前爪世界坐标转换到 `PositionRig` 局部空间，因此宠物缩放、位移和动作变化后仍能保持对齐。

## 吃饭动作

- 头部增加嘴部锚点。
- 饭盆横向位置与嘴部锚点居中，纵深位置取嘴部向前的轻微偏移，垂直位置固定在地面。
- 饭盆每帧根据嘴部锚点更新，避免宠物动作或缩放后出现偏左、偏右的问题。
- 保留食物逐渐减少、低头与咀嚼表现。

## 伸懒腰动作

- 总时长由 3.3 秒延长到 5.4 秒。
- 增加明确的进入、保持、退出阶段，最大姿态保持约 2 秒。
- 前爪向前下方伸出并略微分开。
- 头部与胸口下压并向前移动，身体横向和纵深方向拉长。
- 后爪保持支撑并略微抬高后半身。
- 尾根、中段、尾尖上扬，阴影同步拉长。
- 使用阻尼恢复默认姿态，避免动作结束时突然跳回。

## 构建方式

本版本清理旧 `.output` 与 `.wxt` 后执行：

```bash
wxt build -b chrome
wxt zip -b chrome
```

生成目录：

```text
apps/extension/.output/chrome-mv3
```

生成压缩包：

```text
apps/extension/.output/novaextension-0.4.1-chrome.zip
```
