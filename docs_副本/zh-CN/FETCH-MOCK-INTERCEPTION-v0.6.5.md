# NOVA v0.6.5：Fetch Mock 拦截修复

## 问题

从请求复制 `GET /api/auth/profile` 生成 Mock 规则后，即使当前网站已开启网络拦截，规则仍显示命中 0 次，网页继续访问真实接口。

根因是页面调用 `fetch('/api/auth/profile')` 时，拦截器把相对 URL 错误地解析到了默认的 `http://localhost/`，丢失了当前页面的 `5188` 端口；同时，规则的“网站范围”被误用于比较接口服务器来源，而不是发起请求的网页来源。

## 修复

- Fetch 相对 URL 统一基于 `location.href` 解析，保留协议、主机和端口。
- Fetch 与 XHR 匹配时显式携带当前页面 `location.origin`。
- `scopeOrigin` 只表示规则获准运行的网页来源；接口可以是同源或跨域。
- 完整 Mock 命中后直接构造响应并返回，不再调用真实 Fetch。
- 页面主世界在接收规则后返回 `CONFIGURED` 回执，Network Lab 会显示“拦截器已同步 N 条规则”。

## 使用确认

重新加载扩展并刷新目标网页后，开启网络拦截。状态应显示“拦截器已同步 1 条规则”。再次触发 `/api/auth/profile` 请求后，规则命中次数应增加，请求详情应标记为 Mock。

已经发生的请求不会被追溯替换；新增或启用规则后需要再次触发接口。
