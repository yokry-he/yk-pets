# NOVA v0.6.2：动作切换与 Mock 编辑修复

## 版本目标

本版本修复网络实验室规则编辑空白、宠物待机光环残留和动作切换不符合预期的问题，同时精简动作菜单。

## 网络实验室

- “新增规则”和“根据已有请求生成规则”现在都在稳定的网络实验室根容器内打开全页编辑器。
- 进入编辑器时统一切换到规则上下文，避免 Side Panel 因根节点替换而显示空白区域。
- 保存、取消和草稿恢复继续使用原有规则应用服务与仓储，不改变 Mock 安全边界。

## 动作精简

动作菜单与自动调度中移除了：转圈、玩耍、玩球、爪尖点点。底层协议仍保持向后兼容，旧状态不会导致扩展崩溃。

## 云朵午睡

- 动作时长由 7.6 秒增加到 14 秒。
- 身体在云面上接近水平平躺，头部和身体位置同步调整。
- 进入和退出继续通过阻尼完成，避免突然翻转。

## 打断规则

1. 鼠标移入仅在完全待机时触发一次问候。
2. 鼠标在其他动作期间移入不会排队，也不会在动作结束后补播。
3. 用户点击新动作会立即终止当前动作。
4. NOVA 先进入 240ms 中性姿态，再播放新动作，使切换可控且不生硬。

## 视觉修复

数据光环只在思考或 Agent 执行状态显示；普通待机和生活动作不再持续显示类似加载中的圆环。
