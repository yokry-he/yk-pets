# NOVA 三级菜单与可扩展注册表设计（v0.3.2）

## 目标

本版本将网页内宠物菜单统一为三个同级模式：

- **功能**：页面审计和审计报告。
- **动作**：控制宠物动作。
- **工程**：连接本地 Agent、生成补丁、应用补丁、项目验证和回滚。

所有环形按钮默认只显示图标，中文名称和说明仅在鼠标悬停或键盘聚焦时通过独立 Tooltip 展示。

## 菜单状态

菜单使用互斥状态：

```ts
type MenuState = 'closed' | 'features' | 'motions' | 'agent'
```

因此同一时间只会出现一组按钮。点击宠物统一打开或关闭菜单；再次打开时保留上次选择的模式。思考气泡独立于菜单，不会因为菜单关闭而消失。

## 注册表

功能、动作和工程菜单项都使用统一数据结构注册。后续新增功能或动作时，不需要继续为每个按钮编写独立模板和 CSS 类。

动作定义包含：

- 标识符
- 中文名称
- 中文说明
- 图标
- 动作时长
- 动作分类
- 是否允许加入闲时轮播

## 分页

每个模式每页最多显示 6 个按钮：

- 1～6 个项目：单页半环。
- 7 个及以上：自动分页。
- 支持页码圆点和鼠标滚轮切换。
- 未来动作超过 12 个时，可以在注册表上增加分类过滤，不需要改变菜单核心结构。

## 工程模式

工程相关功能不再使用网页内大型面板。点击图标后直接打开右侧 Side Panel，并将对应操作作为待处理动作传递给侧边栏。

需要 Diff、验证输出、连接配置或确认写入的流程统一在 Side Panel 中完成，避免大面积遮挡宿主网页。

## Tooltip

Tooltip 独立渲染在宠物悬浮层顶层，不再放在圆形按钮内部：

- 中文横向显示。
- 自动在按钮上方或下方放置。
- 不参与按钮布局。
- 不响应鼠标事件，避免闪烁。
- 点击按钮或关闭菜单时立即隐藏。

## 安全规则

- 工程模式中的写入、验证和回滚仍由 Side Panel 和本地 Agent 处理。
- 网页内按钮只发送受约束的 `NovaPetAction`。
- 应用补丁和回滚不会因为改成图标入口而跳过原有确认及安全校验。
