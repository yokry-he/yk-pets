# NOVA v0.6.7：审计范围、网络分类与宠物反馈修复

## 版本目标

v0.6.7 集中关闭 v0.6.6 稳定阶段记录的六项问题，覆盖 Side Panel 任务状态、页面审计范围、Network 请求分类、动作气泡、内置语音和尾尖可见性。

## 任务状态

- 网络实验室、完整报告和 Agent 设置被定义为导航动作，不再写入任务忙碌状态。
- Side Panel 消费导航动作后明确同步 `busy: false`。
- 3D 数据光环只由真实任务状态驱动，普通 `thinking` 动作不会产生加载圈。
- 审计、补丁和检查任务仍在成功、失败与异常路径中复位。

## 页面审计范围

- 审计范围提供性能、无障碍、SEO、最佳实践与 DOM / 页面结构五类多选框。
- 未勾选分类会在规则执行前跳过，不会先执行后隐藏。
- 报告保存 `enabledCategories`，评分、问题总数和严重程度统计只基于本次范围。
- 报告区分“已检查”和“未检查”，并支持类别与严重程度组合筛选。
- 用户最近一次范围保存在 Chrome Storage；宠物双击审计复用同一范围。
- 全部分类未选中时不会启动空审计。

## Network 请求分类

请求页新增带实时数量的分类栏：全部、Fetch/XHR、文档、CSS、JS、字体、图片、媒体、Manifest、Socket、Wasm 与其他。

- 类型筛选可与文本搜索、方法/状态文本和慢请求筛选组合。
- 文档导航通过 Navigation Timing 记录。
- WebSocket 记录连接建立或失败，不读取和修改消息帧。
- Manifest 与 Wasm 结合请求来源、URL 和响应类型归一化。
- 请求详情显示最终资源类型；未知类型稳定进入“其他”。

## 轻量动作气泡与内置语音

- 动作文案不再复用大型审计通知，而是显示在宠物头部附近的小气泡中。
- 小气泡最多两行、自动消失，并使用 `pointer-events: none`，不会阻挡菜单或网页操作。
- 18 个保留动作各自绑定一条内置中文 MP3，与屏幕文案语义一致。
- 内置语音随扩展离线发布，运行时不连接在线 TTS。
- 只对手动动作播放语音；自动闲时动作保持静音。
- 重复当前动作不重播；新动作会停止旧语音；播放失败保持静音，不再回退到提示音。

## 尾尖发光

- 尾尖使用更明亮的青白色，并设置持续生效的基础自发光和外层光晕。
- 尾光流动、甩尾旋风、能量爆发、星星杂耍和烟花动作期间，尾尖在多种颜色间连续闪烁。
- 减少动态效果模式下不执行高频多色变化，保留稳定的明亮尾尖。

## 兼容性

- 旧审计报告没有范围字段时按“全部分类已检查”读取。
- 现有 Network 规则与已保存请求结构保持兼容，新资源类型只扩展枚举。
- 没有增加远程运行时服务、密钥或新的 Chrome 权限。
