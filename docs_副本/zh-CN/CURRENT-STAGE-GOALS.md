# NOVA 当前阶段目标

## 当前基线

- 当前版本：`v0.6.10`。
- 当前阶段：产品能力稳定与缺陷修复。
- 当前实现仍以浏览器扩展为主要交付形态。
- 暂不开始 npm SDK、Web Component 和多框架适配重构。

## 本阶段目标

在不扩大架构改造范围的前提下，提高现有功能的正确性、稳定性和可验证性，为后续平台化拆分建立可靠基线。

优先处理范围：

1. Network Lab 请求采集、规则创建、匹配、Mock 和真实响应修改。
2. Fetch/XHR 拦截一致性、规则热同步和命中状态反馈。
3. 宠物动作中断、过渡、持续时间、悬停隔离、内置语音与视觉反馈。
4. Side Panel 错误边界、状态恢复和窄屏交互。
5. 已发现问题的回归测试、类型检查和生产构建验证。

## 本阶段约束

- 每个修复都必须覆盖实际运行链路，不能只修改界面表现。
- 保持现有规则和本地数据向后兼容；需要迁移时提供安全回退。
- 不在修复过程中提前拆分框架、替换渲染器或改变公共目录结构。
- 不把 Chrome API、Vue、Three.js 或 Agent Provider 的进一步耦合引入领域层。
- 每个正式交付版本统一更新版本号、专项回归、验证报告和构建产物。

## 阶段完成标准

满足以下条件后，当前阶段结束并进入 `v0.7.0`：

- 已知的核心交互、动画和 Network Lab 缺陷已关闭。
- 手动新增、从请求生成、编辑、复制、启停和删除规则稳定可用。
- Fetch 与 XHR 的 Mock、延迟和完整 JSON 响应替换行为一致。
- 页面刷新、扩展重载和 Side Panel 重开后状态可预测。
- 关键路径具备自动化回归，全部工作区类型检查与生产构建通过。
- 现有浏览器扩展可作为平台化重构前的稳定参照实现。

## 下一阶段

`v0.7.0` 将进入 SDK 基础架构阶段，主要工作包括：

- 提取框架无关的宠物核心状态与动作运行时。
- 定义稳定的 `NovaPet` 生命周期、事件和能力接口。
- 将 3D 渲染、Agent、主题、多语言和扩展能力拆为可组合模块。
- 提供原生 JavaScript 和 Web Component 的首批独立接入方式。
- 让现有浏览器扩展逐步改为消费新的核心 SDK。

## 最终目标

项目最终发展为可独立发布的数字宠物 Agent 平台：能够通过 npm、CDN、Web Component 和主流框架直接接入，并支持可替换角色皮肤、多语言、Agent Provider、自定义工具与可选业务插件。

在当前缺陷修复阶段完成前，本节仅作为方向记录，不作为立即实施范围。
