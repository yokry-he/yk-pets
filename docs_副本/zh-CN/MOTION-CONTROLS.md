# NOVA 动作控制与闲时轮播设计

## 目标

v0.2.5 将“网页功能”和“宠物表演”拆成两个明确模式，解决状态气泡、环形按钮和宠物身体互相遮挡的问题，同时允许用户直接控制 NOVA 的动作。

## 分层布局

桌面端使用 440 × 500 像素的透明交互坐标系：

1. 状态气泡固定在最上层区域。
2. “功能 / 动作”模式切换器位于气泡下方。
3. 环形按钮围绕宠物左侧和上侧展开。
4. 宠物固定在右下区域。
5. 工程工具使用独立面板，不与环形按钮同时显示。

状态气泡底部和最上方按钮之间保留至少 12 像素安全间距。按钮轨道也不会覆盖宠物脸部和身体中心。

## 直接动作控制

切换到“动作”模式后提供六个按钮：

| 动作 | Behavior | 主要表现 |
|---|---|---|
| 招手 | `greeting` | 抬起右爪、头部轻摆、开心表情 |
| 跳跃 | `jumping` | 蓄力、起跳、拉伸、落地压缩 |
| 转圈 | `spinning` | 身体绕 Y 轴旋转、尾巴保持平衡 |
| 扑腾 | `flapping` | 两只前爪交替扑腾、耳朵快速响应 |
| 玩耍 | `playing` | 横向移动、身体摇摆、快速摆尾 |
| 休息 | `resting` | 身体压低、头部下降、尾巴收拢 |

动作按钮只修改宠物的受限行为状态，不触发网页审计、DOM 修改或本地项目操作。

## 闲时自动轮播

当满足以下全部条件时启动闲时轮播：

- 功能菜单关闭；
- 页面处于可见状态；
- NOVA 没有执行审计、预览、补丁或项目验证；
- 用户在 9 至 17 秒内没有与宠物互动。

轮播动作从招手、跳跃、转圈、扑腾和玩耍中随机选择，并避免连续播放相同动作。休息动作只通过用户按钮触发，避免宠物长时间保持趴下状态。

以下情况会立即暂停或重置轮播计时器：

- 打开功能菜单；
- 点击任意功能或动作按钮；
- 拖拽宠物；
- 页面切换到后台标签页；
- 审计或 Local Agent 进入忙碌状态。

## 状态优先级

行为显示优先级为：

1. 审计与 Agent 忙碌状态；
2. 用户手动触发动作；
3. 闲时自动动作；
4. 页面健康状态对应的默认动作。

忙碌状态开始时会清除手动和闲时动作，避免宠物一边“运行构建”一边转圈或玩耍。

## 无障碍与减少动态效果

所有动作按钮都使用原生 `button`，可通过键盘触发。开启 `prefers-reduced-motion` 时，CSS 星点、轨道入场和能量波纹会停止动画；3D 宠物仍保留基础状态反馈，但不应持续自动播放高频动作。
