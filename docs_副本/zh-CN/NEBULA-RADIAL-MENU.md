# 星云透明层与环形功能菜单设计

## 1. 目标

v0.2.2 将网页右下角的 NOVA 从“宠物旁边出现一块工具面板”调整为“功能围绕宠物展开”。视觉中心始终是云狐，功能入口作为围绕它运行的轨道节点出现。

设计同时解决四个问题：

- 降低深色矩形面板对宿主页面的遮挡。
- 让云狐展示完整身体、尾巴、浮空底座和光环。
- 通过星云、星点和能量波纹强化生命感，而不引入大图片资源。
- 确保临时预览能够可靠撤销，即使用户在预览后切换了当前问题。

## 2. 透明星云层

云狐 Canvas 保持透明。背景由 CSS 径向渐变组成，并从宠物中心向外逐步衰减到完全透明：

```text
宠物主体
  ↓
近场白色柔光
  ↓
紫色与青色星云
  ↓
低透明雾化边缘
  ↓
完全透明
```

星云层满足以下约束：

- `pointer-events: none`，不会拦截宿主页面操作。
- 不创建新的 WebGL 场景或纹理资源。
- 菜单展开时只提升透明度和范围，不突然显示不透明底板。
- `prefers-reduced-motion: reduce` 下停止星点漂移和脉冲动画。

## 3. 完整身体取景

Compact 模式相机调整为更低的 Y 坐标和更远的 Z 坐标，并适度扩大 FOV。目标取景范围包含：

- 耳朵顶部
- 完整头部与身体
- 两只前爪
- 尾巴末端
- 浮空底座
- 数据光环

移动端使用较小容器，但保持同一完整取景，而不是裁切为头像。

## 4. 半环快捷菜单

桌面端的六个高频功能沿云狐左上方的半圆轨道展开：

1. 页面审计
2. 上一个问题
3. 下一个问题
4. 预览修复 / 撤销预览
5. 详细报告
6. 工程工具

按钮使用圆形轨道节点，不使用固定矩形菜单。节点通过绝对定位形成半环，并按顺序播放轻量展开动画。

窄屏或低高度视口不强行维持环形布局，而是自动切换为云狐上方的三列网格，以避免按钮超出视口。

## 5. 第二层工程工具

点击“工程工具”后，主快捷环收起，出现半透明胶囊工具轨道：

- 连接本地 Agent
- 生成源码补丁
- 确认写入补丁
- 运行项目验证
- 回滚最近补丁
- 返回快捷环

所有涉及源码写入的操作仍然遵守原有确认、哈希校验、备份和回滚规则。

## 6. 能量交互反馈

单击宠物或任何动物功能按钮时，从云狐中心生成一次能量波纹：

- 起始尺寸约为宠物中心区域的 45%。
- 在约 620ms 内扩大到 180%。
- 透明度逐渐衰减到零。
- 动画结束后删除节点，不持续占用 DOM。

星点仅用于环境呼吸感；能量波纹用于明确表达“指令已经被动物接收”。

## 7. 撤销预览保障

临时预览的撤销流程采用三级兜底：

1. 当前问题 ID 存在快照时，恢复该快照。
2. 当前问题 ID 已变化但仍有活动快照时，恢复所有活动快照。
3. 没有活动快照时，显示“当前没有需要撤销的临时预览”。

应用新预览前，会先撤销旧预览，确保页面同一时间最多存在一份 NOVA 临时修改，避免预览叠加导致恢复顺序不确定。

## 8. 可访问性

- 每个环形按钮都有独立 `aria-label`。
- 双击和右键只是快捷方式，所有功能仍有可点击按钮入口。
- 键盘焦点状态不会依赖颜色变化，按钮会同时显示边框和发光反馈。
- 在减少动态效果模式下，按钮直接出现，不播放轨道动画。

## 9. WebGL 透明清屏双保险

v0.2.4 不再仅依赖 Canvas CSS 背景透明，而是同时执行两层设置：

- 向 `TresCanvas` 传入 `clear-alpha="0"`。
- 在 `ready` 事件中再次调用 Renderer 的 `setClearColor(0x000000, 0)`。

此前使用的 `created` 事件并不是当前 TresCanvas 的有效就绪事件，因此回调没有执行，导致部分环境仍显示黑色清屏矩形。改用 `ready` 后，透明清屏会在 Renderer 可用时可靠生效。
