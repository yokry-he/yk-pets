## 本阶段覆盖范围

### 1. 声明式补丁计划

- 使用 `yk-pets.patch-plan/v1` 描述创建、更新、删除和移动操作。
- 更新、删除和移动必须绑定 SHA-256 前置条件。
- 文本编辑按确定顺序执行，并拒绝重叠范围。
- 拒绝绝对路径、目录穿越、反斜杠、`.git`、`.yk-pets/approvals` 和 `node_modules`。

### 2. 精确范围审批

- HMAC 签名的一次性 `yk-pets.scope-approval/v1` 令牌。
- 绑定计划摘要、用户、项目、Origin、项目版本、操作类型、路径集合、文件数量、写入字节预算和有效期。
- 令牌消费后防止重放；范围或计划变化会导致拒绝。

### 3. 文件事务与自动回滚

- 写入和删除通过内容哈希 Compare-And-Swap 执行。
- 事务中途失败时，按逆序自动恢复已应用操作。
- 符号链接和目录不会被跟随或替换。
- 回滚发现外部并发修改时返回冲突，不会强制覆盖。
- 敏感回滚日志可通过公开摘要函数脱敏后进入 UI 或遥测。

### 4. Background / CI 宿主协议

- 使用固定版本的 `yk-pets.workspace-host/v1` RPC。
- 只暴露项目版本、读取、写入和删除等固定命令。
- 不暴露 Shell、动态方法调用、任意代码执行、`Runtime.evaluate` 或脚本注入。

### 5. 修改编排与验证

- `RemediationRunner` 串联预览、审批、事务、验证与回滚。
- 必需验证缺少 Adapter 时执行回滚。
- 验证失败、异常、超时或取消时执行回滚。
- 回滚使用独立信号，不会因调用方已取消而跳过恢复。
- 审计接收器异常不会让已提交事务停留在未知状态。

### 6. 扩展运行时接入

- 新增延迟功能 `safe-modification`。
- 新增受控消息 `modification:run`。
- `authorizeModification` 与页面审计权限分离。
- 未授权时不会下载或初始化修改执行 Bundle。

## 已验证的异常路径

- 路径穿越和受保护目录拒绝。
- SHA-256 前置条件不匹配拒绝。
- 过期、范围不匹配和重复使用的审批令牌拒绝。
- 多文件事务中途失败后逆序回滚。
- 回滚阶段检测并发写入且不覆盖外部内容。
- 验证失败、异常、超时、取消和缺失 Adapter 后回滚。
- 扩展授权拒绝时不加载修改模块。

## 验证边界

本次验证使用 SDK 自动测试、内存文件适配器、严格 RPC 回环适配器和模拟验证宿主。当前运行环境没有用户的真实 Git 仓库、浏览器扩展安装实例、Chrome 调试授权或 CI 凭据，因此没有声称已经在真实项目中创建提交、推送分支、运行真实 Lighthouse/Playwright 浏览器任务或修改生产文件。

在真实项目接入时，应继续执行：

1. 在独立分支或临时工作树中运行；
2. 将 HMAC 密钥限制在可信 Background/本地宿主；
3. 将 Host 授权绑定到扩展发送者、登录用户和项目根目录；
4. 在提交前执行项目自身的 lint、typecheck、test、build 和端到端验证；
5. 保存事务 ID、公开变更摘要和必要的受控回滚日志。
